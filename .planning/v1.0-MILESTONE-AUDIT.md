## BLOCKERS (must close before v1.0 ships)

### G-1: Lobby → EXPLORATION never starts the orchestrator

**Affected:** entire EXPLORE-01..07 + COMBAT-01..12 gameplay loop on a fresh game

`src/eldritch_dm/bot/dynamic_items.py:325-376` — `ReadyButton.callback`'s all-ready branch transitions the channel state but never invokes `orchestrator.start_orchestrator_for_channel`. The orchestrator only starts via `setup_hook`'s RESUME loop (`bot.py:343`), which fires only on bot restart against existing EXPLORATION/COMBAT rows. A fresh `/start_game → all-ready` leaves the channel in EXPLORATION with no orchestrator task running.

**Masked by tests:** Every gameplay integration test constructs the orchestrator directly or routes through the RESUME path. No test covers the cold-start lobby→ready→narrative flow within a single bot lifetime.

**Fix:** 5 lines. Add after `set_state(EXPLORATION)`:
```python
await interaction.client.orchestrator.start_orchestrator_for_channel(
    channel_id_str, session.campaign_name, session.claudmaster_session_id or ""
)
```

### G-2: SanitizerAuditRepo is never instantiated

**Affected:** SAN-05 (sanitizer audit trail)

`SanitizerAuditRepo` exists as a class but no file constructs it at runtime. `exploration.py:107` calls `sanitize_player_input(...)` without `audit_callback=`. Strip events log via structlog but never reach the database. Tests pass because they inject synthetic callbacks directly.

**Fix:** 10 lines. Instantiate in `setup_hook`, expose as `bot.sanitizer_audit_repo`, pass `make_async_audit_callback(...)` into the sanitize call.
