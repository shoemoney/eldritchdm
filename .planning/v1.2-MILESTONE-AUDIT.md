## Cross-Phase Integration

The 3 phases form a tight dependency chain — each verifies its upstream link:

- ✓ **Phase 11 → Phase 12**: Judge call uses Phase 11's `traced_decision` per D-81; spans emit `eldritch.eval.scenario_id`, `eldritch.eval.judge_model`, `eldritch.eval.driver_model`, `eldritch.eval.archetype`. Verified by `tests/observability/test_traced_eval.py`.
- ✓ **Phase 11 → Phase 13**: KPI monitors read from Phase 11's span buffer; Prometheus exporter consumes the same span attributes. Verified by `tests/observability/test_kpi.py` running against synthetic spans matching D-65 schema.
- ✓ **Phase 12 → Phase 13**: MON-01's `eldritch_smart_driver_tactical_score` KPI gauge can be fed by Phase 12's JudgeVerdict aggregates (per D-85). Wire is in place; emits `nan` if eval hasn't been run yet, real value once an eval run has populated the span buffer.
- ✓ **Phase 10 → Phase 11**: SmartMonsterDriver's `_pick_target` wrapped with `traced_decision`; original 51 smart-driver tests still pass after instrumentation (no behavior change — purely observation layer).
- ✓ **Phase 8 → Phases 12/13**: YAML 3-tier loader pattern reused for `database/alerts.yaml` + `database/pricing.yaml`; safe_load CI gate (`scripts/ci/check_safe_yaml.sh`) still passes.
- ✓ **Phase 11 → Phase 13 cost calculator**: token counts flow from OTel spans into cost calculator; ±5% accuracy verified across 5 workloads.

E2E flows verified:
- **Flow D**: Operator enables `OBSERVABILITY_ENABLED=true` + `OBSERVABILITY_METRICS_ENDPOINT=true` → bot emits spans → KPI monitors compute → Prometheus scrape returns live values → 5 KPIs visible
- **Flow E**: Latency injection (P99 > 1500ms for 5min) → DegradedModeState trips → factory returns random MonsterDriver → latency recovery (P99 < 1200ms for 5min) → DegradedModeState clears → factory returns smart again
- **Flow F**: `eldritch-dm-eval --baseline prior.json` runs corpus → judge scores → aggregator diffs → exit 0 if avg held, 1 if regressed > 0.05, 2 if critical dimension < 0.5
