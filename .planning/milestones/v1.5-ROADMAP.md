## Core Goal

Multi-level cache across the stack — dm20 MCP query cache (rules
lookups hot path), persistent character cache (kill restart-latency
UX issue), opt-in narration cache (cost reduction with hard
mechanical-honesty guardrails).

## Key Accomplishments

1. **dm20 MCP query cache (Phase 16)** — L1 in-process LRU + L2 aiosqlite WAL (opt-in). PYTHONHASHSEED-stable SHA-256 args_hash. Fail-CLOSED allow-list of **6 truly-static reference tools** (advisor-reviewed correction: original proposed list contained mutable-state reads that would have violated v1.0 mechanical-honesty). Auto-invalidation on dm20 schema-version change via 60s background poller.
2. **Persistent character cache (Phase 17)** — CharacterCacheRepo with 14-field static-only allow-list (id, name, race, class, subclass, max_hp, base_stats, level, proficiency_bonus, alignment, languages, base_ac, speed, equipment). Synthetic SHA-256 ETag (dm20 has no HTTP headers — primary savings from TTL short-circuit, documented honestly). `eldritch-dm-cache-clear --characters` CLI on PATH.
3. **Narration response cache (Phase 18)** — Opt-in (`NARRCACHE_ENABLED=false` default). NarrCacheGate with 8 regex patterns; 50-scenario Apache-2.0 corpus (25 cacheable + 25 non-cacheable) verifies **0% false-negative + 0% false-positive** classification. Ships as standalone API since no current call site is free-form narration (all are JSON-mode); ready to wrap future narration generator. `eldritch-dm-cache-disable --narration` + `eldritch-dm-cache-stats --narration` CLIs on PATH.
4. **Phase 13 cost calculator tie-in** — narration cache savings_usd KPI uses Phase 13's cost calculator. Confirms cache pays for itself.
5. **Honest-report contract delivered 3×** — agents caught: (16) my fictitious-tools-with-mutable-reads allow-list, (17) the ETag-vs-TTL savings narrative, (18) the no-narration-site-yet integration site. All advisor-reviewed before commit.
6. **276 new tests** across v1.5 (40 + 61 + 175). 8/8 import-linter contracts kept throughout.

## Closing Notes

v1.5 demonstrates that the cache-architecture surface can be built
safely IF every layer fail-CLOSES on uncertainty. The mechanical-honesty
contract from v1.0 propagates cleanly through all 3 caches because each
one has an explicit allow-list (Phase 16: 6 tools, Phase 17: 14 fields,
Phase 18: regex gate). Anything not in the allow-list bypasses → real
data integrity, not optimistic optimization.

The honest-report contract continued delivering corrections at the right
time. Three phases, three distinct corrections — all caught before code
shipped.
