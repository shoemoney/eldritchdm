autonomous-mode honest-report contract repeatedly:

## Phase-by-Phase Verification

### Phase 16 — dm20 MCP query cache
- **Status:** complete (2 plans, 11 tasks, 40 new tests)
- **Critical correction by agent**: my proposed allow-list of 13 tools (`dm20__get_character`, `dm20__get_party`, etc.) was rejected as containing fictitious tool names AND mutable-state reads. Final allow-list: `dm20__get_class_info`, `dm20__get_race_info`, `dm20__list_campaigns`, `dm20__get_campaign_info`, `dnd__search_all_categories`, `dnd__verify_with_api` — 6 tools, all truly static reference data.
- **Tooling clean:** ruff + lint-imports 8/8 contracts kept

### Phase 17 — Persistent character cache
- **Status:** complete (2 plans, 4 atomic commits, 61 new tests — 52 character_cache + 7 cache_clear + 2 config)
- **CharacterSnapshot allow-list**: 14 static-only fields (id, name, race, class, subclass, max_hp, base_stats, level, proficiency_bonus, alignment, languages, base_ac, speed, equipment); 11 forbidden fields parametrized-test-rejected (current_hp, current_conditions, current_temp_hp, etc.)
- **Synthetic SHA-256 ETag**: documented as PRIMARY (not fallback) since dm20 MCP surface has no HTTP headers; network savings come from TTL short-circuit
- **CLI**: `eldritch-dm-cache-clear --characters [--character-id ID]` on PATH

### Phase 18 — Narration response cache
- **Status:** complete (2 plans, 11 commits, 175 new tests across 8 files)
- **NarrCacheGate**: 8 regex patterns covering HP/AC/damage/dice/saves/conditions/sentinel tokens; fail-CLOSED on first match
- **50-scenario corpus**: 25 cacheable + 25 non-cacheable, Apache-2.0 original content; gate classification matches expected on ALL 50 — 0% false-negative + 0% false-positive
- **Honest deviation**: NarrCache NOT wired into existing AsyncOpenAI call sites because none are free-form narration (all use response_format=json_object). Ships as standalone public API ready to wrap a future narration generator. Advisor-reviewed; documented in 18-01-SUMMARY, 18-VERIFICATION.
