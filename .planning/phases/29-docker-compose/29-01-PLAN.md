# Plan 29-01 — docker-compose.yml + multi-stage Dockerfile + .dockerignore

## Objective
Ship the three runtime container artifacts that let a self-hoster run `docker compose up -d` from the repo root and get the `eldritch-bot` Discord process running against an external oMLX + dm20 MCP.

## Context
@.planning/phases/29-docker-compose/29-CONTEXT.md (D-221..D-228)

## Tasks

<task id="1" type="auto">
  <what>Create `.dockerignore` at repo root</what>
  <files>.dockerignore</files>
  <behavior>
    Exclude paths irrelevant to the runtime image:
      - VCS: `.git/`, `.gitignore`, `.gitattributes`
      - venvs: `.venv/`, `venv/`, `env/`
      - planning + agent state: `.planning/`, `.claude/`, `.agents/`, `.cursor/`
      - python build artifacts: `__pycache__/`, `*.py[cod]`, `*.egg-info/`, `build/`, `dist/`
      - test artifacts: `tests/`, `.pytest_cache/`, `.coverage`, `htmlcov/`
      - local state: `*.sqlite3`, `*.sqlite3-*`, `eldritch_dm.db*`, `data/`
      - mac noise: `.DS_Store`
      - secrets (defense in depth, not trust): `.env`, `.env.*` (keep `!.env.example`)
      - docs that don't ship in image: `*.md` except top-level README.md
  </behavior>
</task>

<task id="2" type="auto">
  <what>Create multi-stage `Dockerfile` at repo root</what>
  <files>Dockerfile</files>
  <behavior>
    Two stages targeting Python 3.11 on Debian slim:

    STAGE 1 (`builder`):
      - `FROM python:3.11-slim AS builder`
      - Install `uv` via the official `ghcr.io/astral-sh/uv:latest` COPY pattern (binary copy — avoids pip/curl bootstrap)
      - Set `UV_LINK_MODE=copy`, `UV_COMPILE_BYTECODE=1`
      - Copy `pyproject.toml` + `uv.lock` first (cache layer)
      - `uv sync --frozen --no-dev --no-install-project` to populate `/app/.venv` from the lockfile
      - Copy the project source (`src/`, `README.md`)
      - `uv sync --frozen --no-dev` to install the project itself into the same venv

    STAGE 2 (`runtime`):
      - `FROM python:3.11-slim AS runtime`
      - Create non-root user `eldritch` with UID/GID 1000 and home `/home/eldritch`
      - Set `PATH=/app/.venv/bin:$PATH`, `PYTHONUNBUFFERED=1`, `PYTHONDONTWRITEBYTECODE=1`
      - Copy `/app/.venv` from `builder` (chown to `eldritch:eldritch`)
      - Copy `src/` and `README.md` from `builder`
      - `WORKDIR /app`, `USER eldritch`
      - Healthcheck: `CMD ["python", "-c", "import eldritch_dm; import sys; sys.exit(0)"]` (interval 30s, timeout 5s, start-period 10s, retries 3)
      - `ENTRYPOINT ["python", "-m", "eldritch_dm.bot"]`

    Notes:
      - NO `[mac-ocr]` extras (macOS-only; would fail Linux build anyway)
      - NO `[observability]` extras (opt-in; lazy-imported per Phase 11 design)
      - NO compiler toolchain in runtime stage (slim image, wheels only via uv)
      - Image size target: <500MB compressed (pyproject deps are pure-Python or have wheels)
  </behavior>
</task>

<task id="3" type="auto">
  <what>Create `docker-compose.yml` at repo root</what>
  <files>docker-compose.yml</files>
  <behavior>
    Compose v2 schema (no `version:` key — modern compose ignores it).

    Single service `eldritch-bot`:
      - `build: .` (and `image: eldritch-dm:local` so subsequent `up -d` is fast)
      - `container_name: eldritch_bot`
      - `restart: unless-stopped`
      - `env_file: .env` (operator copies `.env.example` → `.env`)
      - `network_mode: host` is NOT used; use the default bridge so the container is isolated. oMLX + dm20 are reached via `host.docker.internal` (operator overrides `MLX_BASE_URL` / `DM20_MCP_URL` in `.env` accordingly, documented as a comment).
      - `extra_hosts: ["host.docker.internal:host-gateway"]` for Linux parity with Docker Desktop semantics
      - Mount a named volume `eldritch-data:/app/data` for the SQLite DB and any persisted state (the bot can `DB_PATH=/app/data/eldritch.sqlite3`)
      - Healthcheck inherited from Dockerfile HEALTHCHECK (compose surfaces it)
      - Top-level comment block explaining: bring-up, tear-down, observability stack reference, and the host.docker.internal note

    Volumes block:
      - `eldritch-data: { driver: local }`

    NO services for oMLX, dm20, or Phoenix. Phoenix lives in
    `docker-compose.observability.yml` (D-221).
  </behavior>
</task>
