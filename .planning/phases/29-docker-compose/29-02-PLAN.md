# Plan 29-02 — Docker compose smoke test script

## Objective
Ship `scripts/ops/test_docker_smoke.sh` so operators can verify their Docker build + compose stack works end-to-end. NOT wired into default CI (assumes Docker dep).

## Context
@.planning/phases/29-docker-compose/29-CONTEXT.md (D-226)

## Tasks

<task id="1" type="auto">
  <what>Create the smoke test script</what>
  <files>scripts/ops/test_docker_smoke.sh</files>
  <behavior>
    POSIX-ish bash script (`#!/usr/bin/env bash`, `set -euo pipefail`).

    Preflight:
      - Verify `docker` is on PATH; otherwise fail with a friendly message.
      - Verify `docker compose version` succeeds (compose v2 plugin); otherwise fail.
      - If `.env` is missing at repo root, create a throwaway `.env.smoke` with a
        placeholder `DISCORD_TOKEN=smoke-test-not-a-real-token` so compose env_file
        resolution doesn't fail. (The smoke test never actually logs into Discord —
        we exec into the container before the bot's main loop runs Discord login.)
        The throwaway file is cleaned up in the trap.

    Smoke cycle:
      1. Build: `docker compose build eldritch-bot`
      2. Up:    `docker compose up -d --no-recreate eldritch-bot`
      3. Wait:  poll `docker inspect` for container status, give up after 60s
      4. Probe: `docker compose exec -T eldritch-bot python -c "import eldritch_dm; print('OK:', eldritch_dm.__name__)"` — assert exit 0 and stdout contains `OK:`
      5. Image-size budget: `docker image inspect eldritch-dm:local --format '{{.Size}}'` — WARN (don't fail) if >500 MiB.
      6. Cleanup (always, even on failure): `docker compose down -v`

    On failure between (2) and (6):
      - Capture `docker compose logs eldritch-bot` to stderr so the operator has triage signal.
      - Exit non-zero.

    Exit codes:
      - 0  smoke pass
      - 1  generic failure
      - 2  docker / compose not installed (preflight)
  </behavior>
</task>
