# Phase 11 Plan 02 — Phoenix stack + dashboards + smoke test

## Objective

Ship the operator-facing pieces of Phase 11: a docker-compose file that brings
up Arize Phoenix + an OTLP collector, three dashboard spec JSON files for
Phoenix to materialize on first boot, an idempotent seed script that creates
the corresponding Phoenix projects, a README "Optional: observability stack"
section, and an integration smoke test that — given a docker daemon — runs 5
mocked combat turns through the SmartMonsterDriver (using the Plan 01
instrumentation) and asserts spans land in Phoenix within 30 seconds.

Plan 01's `init_tracing()` is wired into the bot's setup_hook here so
`OBSERVABILITY_ENABLED=true` actually emits spans during real bot startup.

## Critical sub-decisions

### D-67a — Dashboard JSON format pivots to "spec, not import"
Phoenix does not publish a stable, importable dashboard JSON schema as of
this writing. The pragmatic shape:

- `database/dashboards/{latency,fallback,cache}.json` are OUR-format spec
  files that document:
  - `name`, `description`, `phoenix_project_name`
  - `attribute_filter` (the OTLP attribute key/value to filter spans by)
  - `metric` (P50/P95/P99 latency / count / hit-rate formula)
  - `time_window` (default 24h)
  - `expected_columns` (what the operator should see in Phoenix UI)
- `scripts/observability/seed-dashboards.sh` POSTs these to Phoenix's
  HTTP API to create Phoenix `Project` entities with matching attribute
  filters. Idempotent: skip if project name exists. Failure to seed is
  WARNING, not ERROR — bot still starts.
- README documents the fallback path: paste the queries documented in each
  JSON's `query_recipe` field into the Phoenix UI manually.

### D-69a — Port discriminator
D-64 default is `http://localhost:6006/v1/traces` (Phoenix unified port).
D-69 maps `:4318` (standard OTLP HTTP port) in docker-compose. Both work
on current Phoenix images. Code default stays at `:6006/v1/traces`. README
documents the `:4318` override for operators on Phoenix images that disable
the unified port.

### D-68a — Smoke test skip semantics
The integration smoke test imports `docker` (optional dep). Use
`pytest.importorskip("docker")` plus `pytest.skip("docker daemon unavailable")`
if `docker.from_env().ping()` raises. CI without docker → silent skip, not
failure. Local dev with docker → real validation.

### Task 3 — Idempotent Phoenix bootstrap script

<task type="auto">
<intent>
`scripts/observability/seed-dashboards.sh` — a POSIX shell script that POSTs
the three `database/dashboards/*.json` files to a running Phoenix instance.
Idempotent: HEAD-checks for existing project names first, skips if present.
Failures are WARNING (non-zero exit code only on argument error, not on
"Phoenix unreachable" — that's expected when the stack isn't up).
</intent>

<files>
- scripts/observability/seed-dashboards.sh
</files>

<behavior>
```bash
#!/usr/bin/env bash
# Seed Phoenix projects from database/dashboards/*.json (Phase 11 / OBS-02).
# Idempotent: skips already-existing project names.
# Phoenix must be running at $PHOENIX_BASE_URL (default http://localhost:6006).
set -euo pipefail
PHOENIX_BASE_URL="${PHOENIX_BASE_URL:-http://localhost:6006}"
DASHBOARDS_DIR="${DASHBOARDS_DIR:-$(dirname "$0")/../../database/dashboards}"

if ! command -v curl >/dev/null 2>&1; then
  echo "WARN: curl not found — cannot seed Phoenix dashboards" >&2
  exit 0
fi
if ! command -v jq >/dev/null 2>&1; then
  echo "WARN: jq not found — cannot parse dashboard JSON; install jq" >&2
  exit 0
fi

# Probe Phoenix reachability (warn-only; don't fail the bot startup)
if ! curl -fsS -m 3 "$PHOENIX_BASE_URL/healthz" >/dev/null 2>&1 \
   && ! curl -fsS -m 3 "$PHOENIX_BASE_URL/" >/dev/null 2>&1; then
  echo "WARN: Phoenix not reachable at $PHOENIX_BASE_URL — skipping dashboard seed" >&2
  exit 0
fi

shopt -s nullglob
seeded=0
skipped=0
for f in "$DASHBOARDS_DIR"/*.json; do
  name="$(jq -r '.phoenix_project_name' "$f")"
  # Phoenix's /v1/projects GET enumerates projects; create via POST if absent.
  existing="$(curl -fsS -m 5 "$PHOENIX_BASE_URL/v1/projects" 2>/dev/null \
              | jq -r --arg n "$name" '.data[]?.name | select(. == $n)' || true)"
  if [ -n "$existing" ]; then
    echo "skip: project '$name' already exists"
    skipped=$((skipped+1))
    continue
  fi
  # Create the project (POST body shape per Phoenix HTTP API)
  curl -fsS -m 5 -X POST "$PHOENIX_BASE_URL/v1/projects" \
       -H "Content-Type: application/json" \
       -d "$(jq -c '{name: .phoenix_project_name, description: .title}' "$f")" >/dev/null \
    && { echo "seeded: $name"; seeded=$((seeded+1)); } \
    || echo "WARN: failed to seed $name — Phoenix API may have changed; paste query_recipe manually" >&2
done
echo "phoenix dashboards: seeded=$seeded skipped=$skipped"
```

Make script executable: `chmod +x scripts/observability/seed-dashboards.sh`.

Why the WARN-on-API-failure: Phoenix's /v1/projects HTTP shape may not be
fully stable. The script's job is best-effort idempotent seeding; the worst
case is the operator pastes the `query_recipe` from the JSON into the UI.
</behavior>

<verify>
`bash -n scripts/observability/seed-dashboards.sh` (syntax check) succeeds.
`scripts/observability/seed-dashboards.sh` runs without error when Phoenix is

```python
"""Phase 11 / OBS-02 smoke test. Skips when docker daemon unavailable."""
import asyncio, os, time
import pytest

@pytest.fixture(scope="module")
def phoenix_running():
    docker = pytest.importorskip("docker")
    try:
        client = docker.from_env()
        client.ping()
    except Exception:
        pytest.skip("docker daemon unavailable")
    # Confirm Phoenix is reachable (operator brought it up, or fixture does)
    import urllib.request, urllib.error
    try:
        urllib.request.urlopen("http://localhost:6006/", timeout=3).read()
    except (urllib.error.URLError, TimeoutError):
        pytest.skip("Phoenix container not running on :6006")
    yield

@pytest.mark.asyncio
async def test_five_combat_turns_emit_five_spans(phoenix_running, monkeypatch):
    monkeypatch.setenv("OBSERVABILITY_ENABLED", "true")
    monkeypatch.setenv("OTEL_EXPORTER_OTLP_ENDPOINT", "http://localhost:6006/v1/traces")

    # Lazy import AFTER env is set, so init_tracing picks up the env.
    pytest.importorskip("opentelemetry.sdk.trace")
    from eldritch_dm.observability.tracer import init_tracing
    from eldritch_dm.observability import traced_decision
    assert init_tracing() is True

    # Simulate 5 decisions
    for i in range(5):
        with traced_decision(monster_id=f"goblin-{i}", channel_id="smoke-ch",
                             combat_round=i+1, driver_path="smart") as span:
            span.set_attribute("eldritch.latency_ms", 400 + i)
            span.set_attribute("eldritch.tokens.input", 100)
            span.set_attribute("eldritch.tokens.output", 20)

    # Force flush so BatchSpanProcessor exports before we poll
    from opentelemetry import trace
    provider = trace.get_tracer_provider()
    if hasattr(provider, "force_flush"):
        provider.force_flush(timeout_millis=5000)

    # Poll Phoenix REST API for at most 30s for our spans
    import urllib.request, json
    deadline = time.monotonic() + 30
    found = 0
    while time.monotonic() < deadline:
        try:
            req = urllib.request.Request(
                "http://localhost:6006/v1/spans?span_name=eldritch.monster.decision&limit=20"
            )
            with urllib.request.urlopen(req, timeout=3) as r:
                payload = json.loads(r.read())
                found = len([s for s in payload.get("data", [])
                             if s.get("attributes", {}).get("eldritch.channel.id") == "smoke-ch"])
                if found >= 5:
                    break
        except Exception:
            pass
        await asyncio.sleep(2)
    assert found >= 5, f"expected ≥5 spans in Phoenix within 30s, got {found}"
```

If Phoenix's /v1/spans HTTP query shape differs in the operator's Phoenix
version, the test will skip-or-fail at the polling step with a clear
assertion message — that's acceptable v1.2 behavior (smoke test reveals
real-world API drift early).
</behavior>
