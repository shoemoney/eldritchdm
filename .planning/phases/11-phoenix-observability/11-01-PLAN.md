# Phase 11 Plan 01 — OTel instrumentation + span schema

## Objective

Ship `src/eldritch_dm/observability/` — a NEW package that exposes
`init_tracing()` (lazy OTel SDK setup) and `traced_decision(...)` /
`traced_translate(...)` helpers that wrap the two AsyncOpenAI call sites in
this codebase: `SmartMonsterDriver._pick_target_llm` and
`ingest.translate.translate_character_sheet`. Span emission is opt-in,
controlled by `OBSERVABILITY_ENABLED` (default `false`). When disabled, NO
OpenTelemetry symbols are imported — verified by a lazy-import canary test
that imports the full bot module tree and asserts `"opentelemetry" not in
sys.modules`.

## Critical sub-decisions

### D-65a — "Narration" reconciliation
CONTEXT/REQUIREMENTS say "bot's narration path." Reality of this codebase:
`dm20` MCP owns true narration internally (the config file documents this
explicitly at L89). The only OTHER AsyncOpenAI call site in our code is
`ingest.translate.translate_character_sheet`. Plain reading wins:
**"narration path" → "every AsyncOpenAI call this codebase makes outside the
smart driver." Concretely: the ingest translator.** Document this in the
SUMMARY so it's not a surprise to downstream Phase 13 monitoring work.

### D-65b — Two span schemas, not one
The D-65 schema lists `eldritch.driver.path` as a `Literal["smart","random","cache","mixed"]`.
Ingest is not a driver decision — forcing it into that enum would pollute the
latency-by-path dashboard. Two distinct span schemas:

1. `eldritch.monster.decision` — full D-65 schema (8 attributes):
   `eldritch.monster.id`, `eldritch.channel.id`, `eldritch.combat.round`,
   `eldritch.driver.path` ∈ `{smart,random,cache,mixed}`, `eldritch.latency_ms`,
   `eldritch.tokens.input`, `eldritch.tokens.output`,
   `eldritch.fallback.reason` ∈ `{None,timeout,json_parse,hallucinated_id,refusal,generic,rate_limit}`.
2. `eldritch.ingest.translate` — subset:
   `eldritch.channel.id` (the ingest channel context), `eldritch.latency_ms`,
   `eldritch.tokens.input`, `eldritch.tokens.output`,
   `eldritch.ingest.parse_error` (bool), `eldritch.ingest.model` (str).

### D-65c — Optional-deps group is minimal
Per advisor: do NOT bundle `opentelemetry-instrumentation-openai`. We
hand-roll spans at the driver boundary (D-66). The optional-deps group is:

```toml
[project.optional-dependencies]
observability = [
  "opentelemetry-api>=1.27,<2.0",
  "opentelemetry-sdk>=1.27,<2.0",
  "opentelemetry-exporter-otlp-proto-http>=1.27,<2.0",
]
```

Conservative floor-only ranges (no `==` pins) — Context7 unavailable in this
session for exact-version verification; floors verified against current PyPI
wheel availability in 2026.

### D-65d — Lazy-import invariant
`from opentelemetry import ...` is approximately 50ms cold-import — material
for self-hosters who don't enable observability. Pattern:

- `eldritch_dm.observability.tracer` module-level: ONLY a `_TRACER` sentinel
  (`Any | None`), a `_NOOP_SPAN` pure-Python context manager, and the public
  function defs. NO `from opentelemetry ...` at module top-level.
- `init_tracing()` body: reads env, returns early if disabled, otherwise does
  the OTel imports INSIDE the function body, builds `TracerProvider`/
  `OTLPSpanExporter`/`BatchSpanProcessor`, sets `_TRACER`.
- `traced_decision()` body: `if _TRACER is None: return _NOOP_SPAN`. No
  `opentelemetry` symbol referenced unless tracing is active.

Lazy-import canary test imports `eldritch_dm.bot` (the integration root —
catches transitive pulls from any submodule) and asserts `"opentelemetry" not
in sys.modules`. NOT just `import smart_monster_driver` (which would pass
spuriously if instrumentation lived elsewhere).

```python
# pure-Python — no opentelemetry imports at module level
from contextlib import contextmanager
from typing import Any, Literal
FallbackReason = Literal["timeout","json_parse","hallucinated_id","refusal","generic","rate_limit"]

_TRACER: Any | None = None  # set by tracer.init_tracing()

class _NoopSpan:
    def set_attribute(self, k: str, v: Any) -> None: ...
    def record_exception(self, exc: BaseException) -> None: ...
    def set_status(self, *args, **kwargs) -> None: ...

@contextmanager
def traced_decision(*, monster_id: str, channel_id: str, combat_round: int, driver_path: str):
    if _TRACER is None:
        yield _NoopSpan()
        return
    # OTel-active path — IMPORT INSIDE function to keep cold-start lazy
    from opentelemetry import trace  # noqa: PLC0415
    span_ctx = _TRACER.start_as_current_span("eldritch.monster.decision")
    with span_ctx as span:
        span.set_attribute("eldritch.monster.id", monster_id)
        span.set_attribute("eldritch.channel.id", channel_id)
        span.set_attribute("eldritch.combat.round", combat_round)
        span.set_attribute("eldritch.driver.path", driver_path)
        yield span

@contextmanager
def traced_translate(*, channel_id: str, model: str):
    if _TRACER is None:
        yield _NoopSpan()
        return
    from opentelemetry import trace  # noqa: PLC0415
    with _TRACER.start_as_current_span("eldritch.ingest.translate") as span:
        span.set_attribute("eldritch.channel.id", channel_id)
        span.set_attribute("eldritch.ingest.model", model)
        yield span
```

Important: `from opentelemetry import trace` MUST be inside the function body,
not module-level. The unused `trace` import inside the function is acceptable
(it's there to populate `sys.modules` only when tracing is on).
</behavior>

### Task 3 — `tracer.init_tracing()` with lazy OTel SDK setup

<task type="auto">

</intent>

<behavior>
```python
"""Lazy OTel TracerProvider setup. Imports OTel only when enabled."""
from __future__ import annotations
import os
from eldritch_dm.observability import instrumentation
from eldritch_dm.logging import get_logger

log = get_logger(__name__)
DEFAULT_ENDPOINT = "http://localhost:6006/v1/traces"
SERVICE_NAME = "eldritch-dm"

def is_enabled() -> bool:
    return os.environ.get("OBSERVABILITY_ENABLED", "false").lower() in ("1","true","yes","on")

def init_tracing() -> bool:
    """Initialize OTel tracing if enabled. Returns True if active, False if disabled.

    Idempotent: calling twice with enabled=True is a no-op after the first call.
    """
    if not is_enabled():
        log.info("observability_disabled")
        return False
    if instrumentation._TRACER is not None:
        return True  # already initialized

    # Lazy imports — only paid when enabled
    from opentelemetry import trace  # noqa: PLC0415
    from opentelemetry.sdk.resources import Resource  # noqa: PLC0415
    from opentelemetry.sdk.trace import TracerProvider  # noqa: PLC0415
    from opentelemetry.sdk.trace.export import BatchSpanProcessor  # noqa: PLC0415
    from opentelemetry.exporter.otlp.proto.http.trace_exporter import OTLPSpanExporter  # noqa: PLC0415

    endpoint = os.environ.get("OTEL_EXPORTER_OTLP_ENDPOINT", DEFAULT_ENDPOINT)
    resource = Resource.create({"service.name": SERVICE_NAME})
    provider = TracerProvider(resource=resource)
    exporter = OTLPSpanExporter(endpoint=endpoint)
    provider.add_span_processor(BatchSpanProcessor(exporter))
    trace.set_tracer_provider(provider)
    instrumentation._TRACER = trace.get_tracer(SERVICE_NAME)
    log.info("observability_enabled", endpoint=endpoint, service=SERVICE_NAME)
    return True
```

</behavior>

<behavior>
Two integration points:

**(a) `_choose_target`:** wrap the whole method in `traced_decision(...)`.
The `driver_path` attribute starts as `route` (one of `random|llm|mixed_random|mixed_llm`)
and is OVERWRITTEN by the inner LLM call when relevant (e.g. cache-hit overrides
to `"cache"`, timeout-fallback overrides to `"random"` AND sets fallback.reason).

**(b) `_pick_target_llm`:** within the existing `try/except` blocks:
- On TimeoutError: `span.set_attribute("eldritch.fallback.reason","timeout")`,
  `span.set_attribute("eldritch.latency_ms", latency_ms)`,
  `span.set_attribute("eldritch.driver.path","random")` (override).
- On generic Exception: same pattern with `fallback.reason="generic"` (or
  `"rate_limit"` if `type(exc).__name__` contains "RateLimit").
- On parse error: `fallback.reason="json_parse"`.
- On hallucinated_id (target_pc_id not in candidate_ids): `fallback.reason="hallucinated_id"`.
- On refusal (empty/no content): `fallback.reason="refusal"`.
- On success: `eldritch.driver.path="smart"`, `eldritch.tokens.input` and
  `eldritch.tokens.output` from `completion.usage.prompt_tokens` /
  `completion.usage.completion_tokens` (defensive `getattr(.., None) or 0`).
- On cache hit: `eldritch.driver.path="cache"`, tokens = 0.

The span tag updates happen via the context manager's `span` handle.
Defensively `import` should NOT be added at module top — the `traced_decision`
import from `eldritch_dm.observability` is fine, but the OTel SDK itself
remains lazy via the patterns in Task 2/3.

Add `tokens_input: int = 0` and `tokens_output: int = 0` locals before the
LLM call so the no-content branch can still call `span.set_attribute(...,0)`.
</behavior>
