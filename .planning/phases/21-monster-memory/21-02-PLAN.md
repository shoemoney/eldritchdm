# Phase 21 Plan 02 — Opt-in persistence + session-close hook (honest gap report)

## Objective

Add an opt-in aiosqlite-backed persistence repo for `MonsterMemory` snapshots, gated by
`MONSTER_MEMORY_PERSIST=true` (default `false`), following the Phase 17 character_cache pattern
(WAL, single-writer, fail-soft). Survives bot restart when enabled. Cleared via
`purge_session(channel_id, session_id)`. The session-close cog wiring is documented as a v1.7
follow-up (Phase 18 honest-report pattern) because no clean session-close hook currently exists in
`bot/cogs/lobby.py`.

## Locked Implementation Decisions

| Tag | Decision | Rationale |
|-----|----------|-----------|
| **L-09** | Repo lives at `src/eldritch_dm/persistence/monster_memory_repo.py`. Class name `MonsterMemoryRepo`. Mirrors `CharacterCacheRepo` ctor shape (`settings`, `path` overrides, lazy `_ensure_conn`, `aclose`). | gameplay→persistence import is allowed by the linter; persistence cannot import gameplay (forbidden). The repo therefore takes/returns plain `dict` snapshots — the Plan 21-01 `MonsterMemory.snapshot_dict()` / `from_snapshot()` boundary is the bridge. |
| **L-10** | Schema (D-161): `CREATE TABLE IF NOT EXISTS monster_memory_entries (channel_id TEXT NOT NULL, session_id TEXT NOT NULL, monster_id TEXT NOT NULL, snapshot_json TEXT NOT NULL, last_updated_ts INTEGER NOT NULL, PRIMARY KEY(channel_id, session_id, monster_id))`. Index on `(channel_id, session_id)` for fast purge. WAL + busy_timeout=5000 + synchronous=NORMAL pragmas (same as character_cache). | Phase 17 proven pattern. |
| **L-11** | Settings additions in `eldritch_dm.config.Settings`: `monster_memory_persist: bool = False` (alias `MONSTER_MEMORY_PERSIST`), `monster_memory_path: str = "~/.eldritch/monster_memory.sqlite"` (alias `MONSTER_MEMORY_PATH`). Default disabled. | D-160 opt-in. |
| **L-12** | Repo API: `async upsert(channel_id, session_id, monster_id, snapshot: dict) -> None`; `async load(channel_id, session_id, monster_id) -> dict | None`; `async load_all_for_session(channel_id, session_id) -> dict[str, dict]` (monster_id → snapshot); `async purge_session(channel_id, session_id) -> int`. All methods wrapped in try/except returning safe defaults (fail-soft per D-165). | Minimal surface for hydrate/flush. |
| **L-13** | `MonsterMemoryRegistry` (from Plan 21-01) gains optional `repo: MonsterMemoryRepo | None = None` ctor arg. When provided: (a) `recall` calls a lazy `await self._hydrate(...)` once per `(channel_id, session_id, monster_id)`; (b) a new `async flush(channel_id, session_id, monster_id) -> None` upserts current state; (c) `async purge_session_async(channel_id, session_id) -> None` calls repo + drops in-memory entries. When `repo is None`, registry behaves exactly as Plan 21-01. | Zero-regression preservation; opt-in surface. |
| **L-14** | Bootstrap wiring: `eldritch_dm.bot.bot` (or wherever `SmartMonsterDriver` is constructed) reads `settings.monster_memory_persist`. If true, constructs `MonsterMemoryRepo`, then `MonsterMemoryRegistry(repo=...)`, then passes that into the factory. Disabled → no repo, no registry → SmartMonsterDriver gets `monster_memory=None`. | Single decision point; opt-in defaults safe. |
| **L-15** | **Honest gap report (Phase 18 pattern)**: the cog-side session-close wiring is NOT shipped in this plan. No `/end_game` command exists in `bot/cogs/lobby.py`; the closest hook is `dm20__close_session` mentioned in CONTEXT but no caller is present in the codebase. We ship: env-gated repo + registry hydration + `purge_session_async` API + a `## Known Gap (v1.7)` section in `21-02-SUMMARY.md` documenting the missing cog hook and exactly where to wire it when an `/end_game` lobby flow lands. | Mechanical honesty — do not fabricate a hook. |
| **L-16** | Persistence flush cadence: the registry does NOT auto-flush per observe_* call (would thrash disk). Plan 21-02 exposes `flush(...)` and a `flush_all() -> int` method; bootstrap registers an `atexit`/`cog_unload` best-effort flush. Documented as best-effort — crashes can lose up to the most recent in-memory state, which is acceptable per "memory is observational, not authoritative." | Avoid disk thrash; match character_cache's TTL-pull approach. |

<task id="3" type="auto">
  <name>Extend MonsterMemoryRegistry with optional repo</name>
  <behavior>
    Update `src/eldritch_dm/gameplay/monster_memory.py` per L-13:
    - `MonsterMemoryRegistry.__init__(self, *, repo=None)` — type-hint repo with a Protocol (avoid
      direct import of persistence repo to keep gameplay decoupled at type level; runtime is fine).
    - `async recall_async(channel_id, session_id, monster_id) -> MonsterMemory`: if repo set and
      key unseen, attempts `await repo.load(...)`; on result, hydrates a MonsterMemory via
      `MonsterMemory.from_snapshot`; caches in the in-memory registry; returns it. On any error,
      falls back to empty MonsterMemory (D-165).
    - Sync `recall(...)` from Plan 21-01 stays as-is — it never touches the repo (no async in sync).
      `SmartMonsterDriver._pick_target_llm` is already async, so we'll prefer `recall_async` there
      when the registry has a repo set, else `recall`.
    - `async flush(channel_id, session_id, monster_id) -> None`: serializes the in-memory
      MonsterMemory via `snapshot_dict()` and calls `await repo.upsert(...)`. No-op if repo None or
      key missing. Fail-soft.
    - `async flush_all() -> int`: flushes every in-memory entry; returns count. Fail-soft.
    - `async purge_session_async(channel_id, session_id) -> int`: calls `repo.purge_session` (if
      repo) then drops in-memory matches. Returns count purged.
  </behavior>
