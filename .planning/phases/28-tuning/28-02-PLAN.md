## Design

**CLI shape (D-218):**
```
eldritch-dm-perf-baseline [--baseline PATH] [--output PATH] [--limit-iterations N]
                          [--skip-cprofile] [--paths LIST]
```

- `--baseline PATH` defaults to `.planning/perf-baseline-v1.9.0.json`
- `--output PATH` defaults to `./perf-runs/` directory; CLI writes
  `perf-{timestamp}-{sha}.json` (the fresh profile) +
  `perf-{timestamp}-{sha}.md` (diff report)
- `--limit-iterations N` overrides profiler iterations (default 100); useful
  for fast local checks
- `--skip-cprofile` skips the cProfile loop (defers to profiler default)
- `--paths LIST` comma-separated subset of hot paths (forwarded to profiler)

**Exit codes (D-218):**
- `0` — all p99s within ±10% of baseline
- `1` — at least one p99 > +10% of baseline (regression)
- `2` — at least one p99 > +25% of baseline (critical regression)

**Diff semantics (advisor guidance):**
- New ops in current run not in baseline → skip (don't fail)
- Missing ops in current run vs baseline → skip (operator may have used `--paths`)
- Per-operation worst case wins (any >+25% → 2; else any >+10% → 1; else 0)

**Module location:** `src/eldritch_dm/tools/perf_baseline.py` (matches
`cost_report.py` / `cache_*` pattern under `tools/`).

**Profiler reuse:** Call `scripts.perf.profile_hot_paths.main(argv=...)` rather
than reimplementing measurement. The CLI orchestrates: (1) invoke profiler
with chosen args + temp output, (2) load baseline + fresh JSON, (3) compute
diff, (4) write report + return exit code.

<task id="1" type="auto" tdd="true">
  <name>perf_diff module: load+diff baselines, compute exit code</name>
  <files>
    - src/eldritch_dm/tools/perf_baseline.py (new — diff module + CLI scaffolding)
    - tests/perf/test_perf_baseline_diff.py (new)
  </files>
  <behavior>
    Pure-function diff: given two `BaselineSchema` instances, returns
    `PerfDiff` containing per-operation deltas (`delta_pct`, `status` ∈
    {"ok","warn","critical","new","missing"}), and a derived `exit_code`.

    Rules:
    - `delta_pct = (current.p99_ms - baseline.p99_ms) / baseline.p99_ms * 100`
      (guarded against baseline p99 == 0 → treat as "ok" if current == 0,
      "critical" otherwise)
    - `status` for matched ops: `delta_pct > 25` → "critical"; `> 10` → "warn";
      else "ok"
    - Ops present in current but not baseline → "new" (informational, doesn't
      affect exit code)
    - Ops present in baseline but not current → "missing" (informational)
    - Exit code: 2 if any "critical"; else 1 if any "warn"; else 0

    Tests (≥5):
    1. matched op within ±10% → status "ok", exit 0
    2. matched op at +15% → status "warn", exit 1
    3. matched op at +30% → status "critical", exit 2
    4. baseline p99 == 0, current > 0 → status "critical"
    5. new op in current → status "new", contributes nothing to exit code
    6. missing op in current → status "missing", contributes nothing
    7. mixed: one critical + one warn → exit 2 (critical wins)
  </behavior>
  <verification>
    `uv run pytest tests/perf/test_perf_baseline_diff.py -q` passes
  </verification>
</task>
