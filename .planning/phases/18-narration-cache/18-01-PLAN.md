# Plan 18-01 — NarrCache + fail-CLOSED NarrCacheGate + 50-scenario corpus

## Objective

Introduce `src/eldritch_dm/observability/narration_cache.py` exposing:

- A `NarrCacheGate` class with a fail-CLOSED `is_pure_narration(text: str) -> bool`
  classifier — returns `True` ONLY if NONE of a hard-coded regex set
  matches. Module-level compiled `re.IGNORECASE` patterns. Short-circuits
  on first match.
- A `NarrCache` class with `acompletion(client, *, model, messages, max_tokens,
  temperature, **kwargs) -> ChatCompletion` — wraps any OpenAI-compatible
  `AsyncOpenAI` client. On HIT (key in L1 + inside TTL + gate accepts the
  cached content): serve cached `ChatCompletion`-shaped object. On MISS:
  call upstream, run gate on the response text, store only if gate
  accepts. L1 = `OrderedDict[str, _L1Entry]` + `asyncio.Lock` + monotonic
  TTL. SHA-256 cache key over `(model, system, user, max_tokens,
  temperature)` (D-132).
- A 50-scenario JSONL corpus at `tests/eval/narration_corpus/corpus.jsonl`
  (Apache-2.0, original) — 25 cacheable + 25 non-cacheable. Each entry has
  `{id, text, expected_cacheable, rationale, category}`. Pydantic-validated
  at load time so malformed entries fail at corpus import, not first assert.
- A per-entry enumerated test that asserts gate classification matches
  `expected_cacheable` for every entry (fail-CLOSED: 0% false-negative is
  the non-negotiable success bar).

## Gate regex set (final, advisor-reviewed)

Compiled once at module load with `re.IGNORECASE`. Each pattern is `\b`-bounded
on BOTH sides where the word is complete; stems use `\b` + `\w*` ONLY where the
trailing suffix is genuine (paralyz**ed**, stunn**ing**, charm**er**).

```python
_GATE_PATTERNS: tuple[re.Pattern[str], ...] = tuple(
    re.compile(p, re.IGNORECASE) for p in (
        # HP / hit points
        r"\b(HP|hit\s*points?)\b",
        r"\bhp\d+\b",
        # AC / armor class
        r"\b(AC|armou?r\s*class)\b",
        # Damage / explicit numeric effects
        r"\b(damage|dmg)\b",
        r"\btakes?\s+\d+\b",
        r"\bdeals?\s+\d+\b",
        r"\b(reduced|drops?|falls?)\s+to\s+\d+\b",
        # Crit / nat-N
        r"\b(critical\s+hit|crit)\b",
        r"\bnatural\s+\d+\b",
        # Saves / DC
        r"\bsaves?\s+against\b",
        r"\bsaving\s+throw\b",
        r"\bDC\s*\d+\b",
        # Condition stems (paralyzed/paralyzing, stunned/stunning, ...)
        r"\b(paralyz|stunn|charm|frighten|grappl|incapacit|petrif|poison|restrain|unconsc)\w*",
        # Condition complete words (\b on both sides; no \w*)
        r"\b(prone|status|condition|invisible)\b",
        # Dice notation + HP assignment
        r"\b\d+d\d+\b",
        r"\b\d+\s*hit\s*dice?\b",
        r"\bhp\s*[=:]\s*\d+\b",
        # Sentinel tokens
        r"<\s*player_action\s*>",
        r"<\s*damage\s*>",
        r"<\s*effect\s*>",
    )
)
```

The split between condition-stems (`paralyz\w*`) and complete-condition words
(`\bprone\b`) is deliberate: `\bprone\w*` would match "**prone**ness" but also
permit the regex engine to swallow word boundary on the right, silently
mis-anchoring future additions.

## NarrCache surface

```python
class NarrCacheMetrics(BaseModel):
    model_config = ConfigDict(extra="forbid", frozen=True)
    hits: int
    misses: int
    rejected_by_gate: int
    bypass_count: int           # NARRCACHE_ENABLED=false OR runtime-disabled
    store_rejected_by_gate: int # cache stores blocked by gate
    size: int

class NarrCache:
    def __init__(self, *, settings: Settings | None = None) -> None: ...
    async def acompletion(
        self,
        client: AsyncOpenAI,
        *,
        model: str,
        messages: list[dict[str, str]],   # exactly system+user
        max_tokens: int,
        temperature: float,
        **kwargs: Any,
    ) -> Any: ...                          # returns ChatCompletion-shaped
    def disable_at_runtime(self) -> None: ...     # D-134 (Plan 18-02)
    def enable_at_runtime(self) -> None: ...      # D-134 (Plan 18-02)
    def metrics_snapshot(self) -> NarrCacheMetrics: ...
    def reset_for_tests(self) -> None: ...
```

Gate is invoked in TWO places:

1. **Pre-store** (every MISS): gate the upstream response BEFORE inserting into L1. If gate rejects → increment `store_rejected_by_gate`, return upstream result uncached.
2. **Pre-serve** (every HIT): gate the cached content AGAIN. If gate now rejects (e.g., regex set tightened in a future release and an old entry now leaks) → invalidate the entry, count as MISS, re-call upstream.

This double-gate is the heart of fail-CLOSED: cached content is re-verified on every serve, NOT trusted because it passed once.
