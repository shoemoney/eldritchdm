# Plan 18-02 — Runtime override + savings observability + cache-stats CLI

## Objective

Wire `NarrCache` into the project's observability and operator-control
surfaces:

1. **Runtime disable/enable singleton** (D-134) — a module-level
   `NarrCacheRuntimeOverride` mirroring the Phase 13 degraded-mode pattern,
   so the operator can flip the cache off without restarting the bot.
2. **Span emission via `traced_narrcache`** — dual-sink (Phase 13 span
   buffer + optional OTel). Spans expose `layer ∈ {bypass, hit, miss,
   gate_reject_store, gate_reject_serve}` and `savings_usd` so downstream
   CLIs can aggregate cost savings.
3. **`eldritch-dm-cache-disable --narration [--enable]` CLI** at
   `src/eldritch_dm/tools/cache_disable.py` — flips the runtime override.
   Default is "disable narration cache"; `--enable` flips it back.
4. **`eldritch-dm-cache-stats --narration [--since DATE] [--format
   json|markdown]` CLI** at `src/eldritch_dm/tools/cache_stats.py` — reads
   the Phase 13 span buffer, aggregates `hit_rate`, `total_calls`,
   `cached_calls`, `rejected_by_gate` (both store + serve), and
   `savings_usd` (computed against Phase 13 pricing for the recorded
   token counts). JSON or human-readable Markdown output.

## Span schema

`eldritch.narrcache.call` (one row per `NarrCache.acompletion` invocation):

| OTel attribute                            | Buffer column           | Notes                                               |
| ----------------------------------------- | ----------------------- | --------------------------------------------------- |
| `eldritch.narrcache.layer`                | `driver_path`           | `bypass\|hit\|miss\|gate_reject_store\|gate_reject_serve` |
| `eldritch.narrcache.model`                | `model`                 | e.g. `"ShoeGPT"`                                    |
| `eldritch.narrcache.size`                 | `combat_round`          | L1 size at call exit                                |
| `eldritch.narrcache.latency_ms`           | `latency_ms`            | total wall-clock incl. upstream                     |
| `eldritch.tokens.input`                   | `tokens_input`          | upstream usage (0 on HIT)                           |
| `eldritch.tokens.output`                  | `tokens_output`         | upstream usage (0 on HIT)                           |
| `eldritch.narrcache.savings_usd`          | `overall_score` (reuse) | float; populated on HIT only via cost calculator    |

Reuse of `overall_score` / `combat_round` mirrors Phase 16/17's approach so the
`BufferRow` schema stays stable (no new columns; `test_span_buffer` canaries
remain green).
