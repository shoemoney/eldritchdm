# Phase 22 Plan 01 — Hot-reload eligibility.yaml + file watcher

## Objective

Operators can edit `eligibility.yaml` while the bot is running and have the
homebrew Riposte eligibility set refresh within ~60s with NO bot restart.
Fail-soft on bad YAML (preserves Phase 8 D-31/D-33 contract).

## Context

- @src/eldritch_dm/gameplay/eligibility_loader.py — Phase 8 loader, NEVER raises
- @src/eldritch_dm/bot/bot.py — `setup_hook` boots loader; `close()` tears down
- @tests/gameplay/test_eligibility_loader.py — Phase 8 black-box tests
- D-167: extend loader with `reload()` + `EligibilityFileWatcher`, 60s mtime poll
- D-168: stdlib-only (`asyncio.create_task`, `pathlib.stat`)
- D-174: any failure logs + continues, never raises into bot loop

## Tasks

### Task 1 — Extend `eligibility_loader.py` with `EligibilityFileWatcher`
<task type="auto">
<files>src/eldritch_dm/gameplay/eligibility_loader.py</files>

<behavior>
Add a new class `EligibilityFileWatcher` to the same module:

```python
class EligibilityFileWatcher:
    def __init__(
        self,
        settings: Settings,
        *,
        initial_set: frozenset[tuple[str, str]] | None = None,
        poll_interval_s: float = 60.0,
        on_reload: Callable[[frozenset[tuple[str, str]]], None] | None = None,
    ) -> None: ...

    @property
    def current(self) -> frozenset[tuple[str, str]]: ...

    async def start(self) -> None: ...    # idempotent
    async def stop(self) -> None: ...     # idempotent

    # Test seam — runs one poll iteration synchronously
    def poll_once(self) -> bool: ...      # True iff reload happened
```

Semantics:
- Path is resolved ONCE at construction via `_resolve_path(settings)`. If
  None at init: watcher still starts but every poll is a no-op (log
  `eldritch.eligibility.watcher_no_path` ONCE on start).
- Captures `path.stat().st_mtime_ns` on first successful read.
- Each poll: stat the path; if file missing → log
  `eldritch.eligibility.watcher_file_missing` ONCE and keep last-known-good.
  Do NOT reset baseline mtime — if file reappears with new mtime, that
  triggers a reload.
- If mtime unchanged → no-op.
- If mtime changed → call `load_eligibility(settings)`. The loader already
  fail-softs to DEFAULT_ELIGIBILITY on bad YAML, so we get either:
    * a new valid frozenset → store, log `eldritch.eligibility.reload_succeeded`,
      invoke `on_reload(new_set)` (errors swallowed)
    * DEFAULT_ELIGIBILITY (fallback path) — distinguishable from a "real" valid
      load only by comparing to `last_known_good`. **Decision:** if loader
      returns `DEFAULT_ELIGIBILITY` AND `last_known_good != DEFAULT_ELIGIBILITY`
      AND the file existed (we just statted it), treat that as "reload failed"
      → log `eldritch.eligibility.reload_failed`, KEEP `last_known_good`,
      do NOT call `on_reload`. Otherwise update baseline mtime + last_known_good.
- `start()`: spawn one `asyncio.create_task(self._run())`. Idempotent.
- `stop()`: cancel + await the task. Idempotent.
- `_run()` loop: `await asyncio.sleep(poll_interval_s)`, then `poll_once()`
  inside a `try/except BaseException` that logs and continues; only
  `asyncio.CancelledError` exits the loop.
- ALL exceptions in `poll_once` are caught + logged
  `eldritch.eligibility.watcher_error` and the function returns False. The
  loop never raises into the event loop.
</behavior>

### Task 3 — Wire watcher lifecycle into `bot.setup_hook` / `bot.close`
<task type="auto">
<files>src/eldritch_dm/bot/bot.py</files>

<behavior>
In `EldritchBot`:

1. Add attribute `self.eligibility_watcher: EligibilityFileWatcher | None = None`
   in `__init__`.
2. In `setup_hook`, AFTER `self.eligibility_set = load_eligibility(settings)`:

```python
from eldritch_dm.gameplay.eligibility_loader import EligibilityFileWatcher

def _apply_new_eligibility(new_set: frozenset[tuple[str, str]]) -> None:
    self.eligibility_set = new_set
    # MonsterDriver holds the OLD frozenset — Phase 8 D-38. Hot-reloading
    # into a live driver is out of scope for OPQOL-01: bot exposes the
    # updated set on self.eligibility_set; future driver instances pick
    # it up on next construction. Document this in the SUMMARY.
    log.info("eligibility_hot_reloaded", count=len(new_set))

self.eligibility_watcher = EligibilityFileWatcher(
    settings,
    initial_set=self.eligibility_set,
    on_reload=_apply_new_eligibility,
)
await self.eligibility_watcher.start()
```

3. In `close()`, BEFORE step 0a (riposte_sweeper stop):

```python
if self.eligibility_watcher is not None:
    try:
        await self.eligibility_watcher.stop()
        self._logger.debug("eligibility_watcher_stopped")
    except Exception:
        self._logger.exception("eligibility_watcher_stop_error")
```

The "live driver still has old set" caveat is the deliberate v1.6 scope cut:
the watcher updates `bot.eligibility_set` so any subsequently-constructed
driver picks it up; rebuilding the live driver mid-combat is out of scope.
This must be called out in the SUMMARY.
</behavior>
