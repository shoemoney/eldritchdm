       - For dm20 MCP cache: `MCPCACHE_ENABLED` defaults to `true` but the mutation/mutable-state allow-list rejects most calls (Phase 16 / D-117). L2 only fires if `MCPCACHE_L2_ENABLED=true`.

Note: `python -m eldritch_dm.bootstrap` and `python run.py --check-only` do NOT require the token (Phase 5 / D-26).

    5. **Pytest hangs in the full suite** — Known orchestrator-session-specific behavior; not a code bug. Restart the shell or run tests in a new pytest invocation. Cite the AI-SPEC.md / Phase 14 milestone note (FLAKE-02 partial — user-accepted tech debt per v1.3 audit).

    6. **Monster driver always picks random targets** — Two causes:
       (a) `MONSTER_DRIVER=random` is explicitly set in `.env` (v1.0 escape hatch — Phase 10 / D-52).
       (b) Degraded mode tripped: when P99 latency exceeds the budget, SmartMonsterDriver falls back to random. Cite Phase 22 OPQOL-02 / D-170 (degraded-mode DMs).
       Fix: unset `MONSTER_DRIVER` (defaults to `smart`); investigate latency via `eldritch-dm-perf-baseline`.

    7. **Riposte button doesn't fire** — The `pc_classes` table isn't populated for the character. Standard v1.0→v1.1 upgrade gap (Phase 9 / TD-3 / UPGRADE-01). Fix: `eldritch-dm-backfill-pc-classes`. See `docs/UPGRADE.md#v10--v11` for the full flow. Caveat: dm20 doesn't expose subclass, so Battle Master Riposte still needs manual `UPDATE pc_classes SET subclass='battle master' …`.

    8. **Phoenix dashboards empty** — Both env vars required:
       - `OBSERVABILITY_ENABLED=true` (`src/eldritch_dm/observability/tracer.py:39`)
       - `OBSERVABILITY_METRICS_ENDPOINT=true` (`src/eldritch_dm/observability/metrics_endpoint.py:51`) for Prometheus `/metrics` on port 9090
       And `pip install -e ".[observability]"` (the optional dep group). Cite Phase 11 OBS-01 and Phase 13 MON-01.

    9. **Cost calculator is off / shows wrong dollars** — `database/pricing.yaml` is stale. Refresh it (v1.2.1 hotfix pattern). Cite Phase 13 MON-03 + `eldritch-dm-cost-report` CLI. Set `ELDRITCH_DAILY_LLM_BUDGET_USD` to your real cap (default `5.00`).

    10. **perf-baseline regression alert** — `eldritch-dm-perf-baseline` exited 1 or 2 (Phase 28 / TUNE-02 / D-218). Exit 1 = regression detected vs. committed baseline; exit 2 = profile-run failure. Fix: investigate hot path; if intentional, regenerate baseline with `--write-baseline .planning/perf-baseline-v1.X.0.json`.

    11. **Restart loses character state** — Two possible causes:
       (a) **Combat state**: SQLite-persisted; should survive (Phase 4 OPS-04 restart-resume). If it doesn't, check `eldritch.sqlite3` is the same file and `PRAGMA journal_mode` returns `wal`.
       (b) **Character snapshot cache**: auto-enabled from v1.5 (Phase 17 / D-119, `CHARCACHE_ENABLED=true` by default). If reading from a stale snapshot, `eldritch-dm-cache-clear` purges it.
       Cite Phase 17 CONTEXT.

    12. **Eligibility YAML edits don't take effect** — Phase 22 OPQOL hot-reload watcher is mtime-polled. Save the file to bump mtime; the watcher logs `eligibility.yaml.reloaded`. Cite Phase 22 OPQOL bundle.

    13. **`sqlite3.OperationalError: database is locked`** (moved from INSTALL.md) — Phase 15 v1.4 writer-queue fix made the writer serial via one connection. If you still see this, another process holds a write lock (REPL, second `python run.py`, SQLite browser). Verify WAL: `sqlite3 eldritch.sqlite3 'PRAGMA journal_mode;'` returns `wal`. Cite `.planning/milestones/v1.4-ROADMAP.md`.

    14. **Bot disconnected mid-combat, restart, buttons inert** (moved from INSTALL.md) — `DynamicItem` regex-template mismatch for stale custom_ids. Cite Phase 4 persistent-view discipline (existing `docs/ARCHITECTURE.md`). Fix: `DELETE FROM persistent_views WHERE …` or wait for the sweeper.
