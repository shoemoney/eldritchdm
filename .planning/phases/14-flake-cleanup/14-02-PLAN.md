## Tasks

### Task 1 — Write the backfill script <type="auto">

**Behavior:** `scripts/audit/backfill_summary_frontmatter.py` takes a hard-coded mapping
(extracted from ROADMAP.md Traceability — embedded in the script for reproducibility),
reads each SUMMARY.md, parses its YAML frontmatter (between the first two `---` lines),
inserts or replaces `requirements_completed: [REQ-ID, ...]` (alphabetically sorted),
removes any legacy `requirements-completed:` line, and writes the file back.

Supports `--dry-run` (default) and `--apply` modes. `--apply` rewrites in place;
dry-run prints a unified diff per file.

**Files:**
- `scripts/audit/backfill_summary_frontmatter.py`

**Verify:** `python scripts/audit/backfill_summary_frontmatter.py --dry-run` →
prints planned changes; exits 0.

### Task 2 — Apply backfill to all 14 SUMMARYs <type="auto">

**Behavior:** Run the script with `--apply`. Each SUMMARY now has the
`requirements_completed:` line in its frontmatter.

**Files:** All 14 SUMMARY.md files listed above.

**Verify:**

### Task 3 — Add CI gate `scripts/ci/check_summary_frontmatter.sh` <type="auto">

**Behavior:** Shell script that:
1. Lists every `*-SUMMARY.md` under `.planning/phases/`.
2. For each, asserts the file starts with `---` (has frontmatter) AND contains a
   `requirements_completed:` key in its frontmatter block.
3. Exits 1 with a clear message if any file fails the check, listing the offenders.

Hook is invoked by CI (or pre-commit) — but for v1.3 we ship the script only;
wiring it into a workflow is post-v1.3 polish.

**Files:**
- `scripts/ci/check_summary_frontmatter.sh`

**Verify:** `bash scripts/ci/check_summary_frontmatter.sh` exits 0 after Task 2.
