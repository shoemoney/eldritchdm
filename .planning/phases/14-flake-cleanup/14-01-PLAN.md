## Tasks

### Task 1 — Fix `test_unsupported_bytes_returns_zero_confidence` (FLAKE-01) <type="auto">

**Behavior:** The test feeds garbage bytes with `content_type="image/png"`. The current
`_sniff_kind` falls through to `content_type` (returns "image"), then the pipeline calls
`resolve_ocr_backend()` which returns None on a dev venv → raises `UnavailableOCRBackend`.

**Fix:** Patch `resolve_ocr_backend` in this test to simulate a backend being available
(mirroring the pattern in `_make_patches`). The test's intent is "unknown magic bytes
yield a zero-confidence result with a warning" — but with `image/png` declared, the
sniff returns "image". The simplest correct fix is to make the test use a content_type
that actually does not match a known kind (e.g. `application/octet-stream`), so the
ValueError-path is exercised. Update test inputs accordingly.

**Files:** `tests/ingest/test_pipeline.py`
**Verify:** `uv run pytest tests/ingest/test_pipeline.py -q` → all pass without
needing `ocrmac` installed.

### Task 2 — Add `pytest.importorskip("prometheus_client")` to test_metrics_endpoint (FLAKE-01) <type="auto">

**Behavior:** `tests/observability/test_metrics_endpoint.py` calls
`start_metrics_endpoint()` which tries `from prometheus_client import …` inside the
function. `prometheus_client` lives in the `observability` extras group — not in dev.
The 4 tests that actually start the endpoint MUST be skipped when the extra is not
installed.

**Fix:** Add a module-level `prometheus_client = pytest.importorskip("prometheus_client")`
(per D-95 pattern). Tests that only check env-gate behaviour (`is_metrics_endpoint_enabled`,
`noop_when_disabled`) can stay; tests that actually call `start_metrics_endpoint()` with
the env enabled must be skipped.

**Files:** `tests/observability/test_metrics_endpoint.py`
**Verify:** `uv run pytest tests/observability/test_metrics_endpoint.py -q` →
4 skipped (clean skip), 3 pass.
