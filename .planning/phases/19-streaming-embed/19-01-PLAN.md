# Phase 19 Plan 01 — Thinking indicator + EmbedCoalescer integration (STREAM-01)

## Objective

Wire an optional `embed_update_callback` through `SmartMonsterDriver` so that
when the LLM oracle is about to be invoked (route ∈ {`llm`, `mixed_llm`}, no
cache hit), the driver emits a single "🤔 {monster_name} is sizing up the
party..." message via a caller-supplied async callback. The callback is
cancellation-safe (D-145): any exception from the callback path is swallowed
so combat continues unconditionally.

## Context

- `SmartMonsterDriver._pick_target_llm` is the LLM call site. Cache-hit short
  circuit must NOT fire the indicator.
- The callback is `Callable[[str, str], Awaitable[None]]` with args
  `(channel_id, message_text)`. The bot cog wires this to its per-channel
  `EmbedCoalescer` (Plan 02). This module remains import-linter-safe — it
  never imports from `bot/`.
- `contextlib.suppress(Exception)` wraps the callback so a closed coalescer
  queue (shutdown scenario) cannot crash combat (D-145, mirrors D-58 fail-soft).

## Tasks

### Task 1 — Add `embed_update_callback` kwarg to SmartMonsterDriver

Add an optional constructor kwarg:

```python
embed_update_callback: Callable[[str, str], Awaitable[None]] | None = None
```

Store as `self._embed_update_callback`. Default `None` preserves backward
compatibility with all 51 existing tests + the 16-scenario corpus.

### Task 2 — Fire indicator before oracle call (cancellation-safe)

In `_pick_target_llm`, AFTER the cache-hit short-circuit but BEFORE the
`asyncio.wait_for(...)` LLM call, invoke the callback if set:

```python
if self._embed_update_callback is not None:
    monster_name = current_actor.get("name", "monster")
    text = f"🤔 {monster_name} is sizing up the party..."
    with contextlib.suppress(Exception):
        await self._embed_update_callback(channel_id, text)
```

NO indicator on:
- random / mixed_random routes (LLM never called)
- cache hits (LLM never called)
- after the LLM call (resolved-action embed is the bot cog's responsibility —
  D-141 unchanged)

### Task 3 — Tests (≥4 in this plan; Plan 02 adds the rest to reach ≥8 total)

In a new `tests/gameplay/test_smart_monster_driver_streaming.py`:

1. `test_thinking_indicator_fires_on_llm_route` — INT≥8 monster, callback set;
   assert callback invoked exactly once with `(channel_id, "🤔 ... sizing up
   the party...")` text.
2. `test_thinking_indicator_no_fire_on_random_route` — INT≤4 monster; assert
   callback NEVER invoked.
3. `test_thinking_indicator_no_fire_on_cache_hit` — pre-populate driver cache;
   second call hits cache; callback NEVER invoked.
4. `test_thinking_indicator_swallows_exception` — callback raises
   RuntimeError; combat still resolves to a valid target (no propagation).
