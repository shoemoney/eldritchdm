# Phase 33 Plan 01 — Final Consolidation (README + CHANGELOG + cross-links)

## Objective

EldritchDM has shipped 11 milestones (v1.0 → v1.11) but the README still claims `v1.0-ready` and no `CHANGELOG.md` exists. This plan brings the user-facing documentation surface up to date with three documentation-only deliverables:

1. **DOCS-04** — README refresh: bump the status badge to `v1.11`, replace the now-stale "v1.1 lookahead" block with a "Recent milestones (v1.1 → v1.11)" condensed table linking to each milestone archive + CHANGELOG, and fold mentions of Phoenix observability / eval CLI / perf CLI / Docker quickstart into their natural existing homes. **Preserve the project-tour tone.**
2. **DOCS-05** — New `CHANGELOG.md` at repo root in [Keep a Changelog](https://keepachangelog.com) format. One entry per shipped tag (v1.0, v1.1, v1.2, v1.2.1, v1.3, v1.4, v1.5, v1.6, v1.7, v1.8, v1.9, v1.10, v1.11) — 3-5 bullet headlines per entry synthesized **only** from existing `.planning/milestones/v1.*-ROADMAP.md` + `.planning/v1.*-MILESTONE-AUDIT.md` content. Cross-link each entry to its archive.
3. **DOCS-06** — Cross-link consistency: add a `CHANGELOG.md` bullet to the "Companion docs" callouts in `INSTALL.md`, `docs/TROUBLESHOOTING.md`, and `docs/UPGRADE.md`. The README must reach all four (INSTALL / CHANGELOG / TROUBLESHOOTING / UPGRADE).

## Constraints

- **Documentation only.** No code, test, or config changes. `ruff` + `lint-imports` are no-op passes.
- **No invented content.** Every CHANGELOG bullet must trace back to a specific line in a milestone archive or audit doc. Traceability captured in `33-VERIFICATION.md`.
- **Do NOT modify `STATE.md` or `ROADMAP.md`** (per phase brief — milestone-completion script owns those).
- Preserve README tone: it's the project tour, not a release-notes dump.
