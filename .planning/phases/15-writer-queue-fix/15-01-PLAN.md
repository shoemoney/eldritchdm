# Phase 15 Plan 01 — FLAKE-02 Closure via Test-Isolation Patch

## Objective

Close FLAKE-02 (carried partial from v1.3; reopened as HANG-03 in v1.4) by
fixing test-isolation in `tests/bot/conftest.py::bot_factory`. The fix is in
test infrastructure only — `src/eldritch_dm/persistence/` is correct as
verified by the prior Phase 15 agent's halt report.

## Diagnosis (from 15-HALT-REPORT.md + this agent's reproduction)

Failure surface: `tests/integration/test_phase3_smoke.py` — two tests fail
in the full suite after any test that fully boots the bot via `bot_factory`
runs first. The failures are `AssertionError: send_kwargs.get("view") is not
None`, downstream of `UnavailableOCRBackend` being raised because the
`mock.patch("eldritch_dm.bot.cogs.ingest.ingest", AsyncMock(...))` is being
bypassed by the cog's already-captured original `ingest` symbol.

Root cause (`tests/bot/conftest.py:bot_factory`):

1. `bot_factory` calls `bot.setup_hook()` which calls
   `await bot.load_extension("eldritch_dm.bot.cogs.ingest")`.
2. discord.py's `load_extension` imports the cog module and caches it in
   `sys.modules`.
3. The cog binds `from eldritch_dm.ingest import ingest` as a module-global
   at import time — the cog's callback closes over that specific function
   reference.
4. `bot.close()` does NOT call `unload_extension`. The cached `sys.modules`
   entry and the cog's bound symbol survive to the next test.
5. Phase3 smoke calls `mock.patch("eldritch_dm.bot.cogs.ingest.ingest",
   AsyncMock(...))`. The patch lands on the cached module dict, but the
   cog's callback already captured the original `ingest` — so the mock
   never sees the call. Real `pipeline.ingest` runs, OCR backend missing,
   `UnavailableOCRBackend` raised, assertion fails.

Why HANG-01 / HANG-02 are NOT in scope: prior agent confirmed they pass in
both isolation and full suite. Phase 14 logging-polluter fix already
neutralized them. Mark complete with a verification note.

## Fix

Convert `bot_factory` from a sync fixture returning an async callable into
an `@pytest_asyncio.fixture async` fixture that:

1. Tracks every bot it constructs in a local list.
2. Yields the factory callable as before.
3. In teardown, iterates `created_bots`; for each, calls
   `await bot.unload_extension(name)` for every name in `bot.extensions`,
   then `await bot.close()` if not already closed.

`unload_extension` does `del sys.modules[name]` (verified by discord.py
source), which clears the stale binding so the next test's `mock.patch`
establishes against a fresh import of the cog module.

The fixture's public contract is unchanged: callers still `await
bot_factory()` to get a constructed bot. The `running_bot` fixture which
calls `await bot.close()` in its own teardown remains correct — the new
`bot_factory` teardown checks `bot.is_closed()` and skips already-closed
bots.
