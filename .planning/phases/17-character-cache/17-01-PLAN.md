# Plan 17-01 — CharacterCacheRepo (snapshot SQLite + synthetic-ETag refresh + static-fields allow-list)

## Objective

Introduce `src/eldritch_dm/persistence/character_cache.py` exposing a `CharacterCacheRepo` class with:

- A pydantic `CharacterSnapshot` model whose `model_config = ConfigDict(extra="forbid", frozen=True)` hard-codes a **static-fields-only allow-list** (D-125). Combat-mutable fields (`current_hp`, `current_temp_hp`, `current_conditions`, etc.) are rejected at write time — fail-CLOSED.
- An aiosqlite WAL store at `~/.eldritch/character_cache.sqlite` (D-119), separate from `mcp_cache.sqlite` (Phase 16) and `eldritch.sqlite3` (Phase 1). Lazy `_ensure_conn()` mirroring `MCPCache._l2_ensure_conn()` — NOT routed through Phase 1 `WriterQueue`.
- A `get_or_fetch(character_id, fetcher)` async API. On HIT (entry exists, ETag matches synthetic SHA-256 of the latest fetch, AND inside TTL): serve cached snapshot in <10ms, bump `last_seen_ts`. On MISS or ETag-mismatch: call the supplied `fetcher`, project the response to the `CharacterSnapshot` allow-list, store, return.
- A synthetic ETag = `sha256(json.dumps(response, sort_keys=True, separators=(',',':'), default=str)).hexdigest()`. dm20 surface is MCP (no HTTP headers) — synthetic is the **primary** path, not a fallback.

TTL fallback, `eldritch-dm-cache-clear` CLI, and KPI emission are deferred to Plan 17-02.

## Locked design decisions (CONTEXT carry-over)

- **D-119** Standalone cache at `~/.eldritch/character_cache.sqlite`. aiosqlite WAL. Lazy connection — NOT WriterQueue.
- **D-120** Schema:
  ```sql
  CREATE TABLE character_cache_entries (
    character_id   TEXT    PRIMARY KEY,
    snapshot_json  TEXT    NOT NULL,
    etag           TEXT    NOT NULL,
    last_seen_ts   INTEGER NOT NULL,
    refreshed_ts   INTEGER NOT NULL
  );
  CREATE INDEX idx_character_cache_refreshed ON character_cache_entries(refreshed_ts);
  ```
- **D-121** Lazy first-load: `get_or_fetch(character_id, fetcher)` populates on demand. No startup preload.
- **D-122** Synthetic ETag = `sha256(canonical-json(latest dm20 response))`. On `get_or_fetch`: re-fetch + compute fresh ETag; if it matches cached → serve cached, bump `last_seen_ts`; else → store the new projection.
- **D-125 (HARD CONSTRAINT — mechanical honesty)** `CharacterSnapshot` allow-list (advisor-reviewed against the D-125 forbidden list):

  **Cacheable static fields (FINAL):**
  - `id: str` — character UUID
  - `name: str`
  - `race: str`
  - `character_class: str` (renamed from `class` — `class` is a Python keyword)
  - `subclass: str` (empty string allowed)
  - `level: int`
  - `proficiency_bonus: int`
  - `alignment: str`
  - `languages: list[str]` (sorted)
  - `max_hp: int` — NEVER changes from damage; only grows on level-up
  - `base_stats: dict[Literal["STR","DEX","CON","INT","WIS","CHA"], int]` — ASI changes only
  - `base_ac: int` — armor + DEX + class features ONLY; NO buffs / spells / conditions
  - `base_speed: int` — racial/class base ONLY; NO grapple / prone / exhaustion modifiers
  - `equipment: list[str]` — items owned, NOT item-derived active effects

  Field names `base_ac` and `base_speed` (not bare `ac`/`speed`) are deliberate — the prefix makes the static-vs-current distinction visible at every call site.

  **Explicitly FORBIDDEN** (D-125, enforced by `extra="forbid"`):
  - `current_hp`, `current_temp_hp`, `current_conditions`, `exhaustion_level`,
    `active_buffs`, `concentration_target`, `death_save_successes`,
    `death_save_failures`, `hit_dice_remaining`, `current_speed`, `current_ac`.

- **D-127** Module at `src/eldritch_dm/persistence/character_cache.py`. Tests at `tests/persistence/test_character_cache.py`.

### T-17-01-03 — `get_or_fetch` with synthetic-ETag refresh path

**Files:** `src/eldritch_dm/persistence/character_cache.py` (extend), `tests/persistence/test_character_cache.py`.

**Behavior:**
- Signature: `async def get_or_fetch(self, character_id: str, fetcher: Callable[[str], Awaitable[dict[str, Any]]]) -> CharacterSnapshot`.
- Semantics:
  1. Read row by `character_id`. If absent → MISS branch.
  2. Always call `fetcher(character_id)` to get the **current** dm20 response. (D-122: dm20 is MCP, not HTTP — there are no real ETag headers, so we must fetch to verify.)
  3. Compute `new_etag = etag_of(response)`. If row existed AND `new_etag == row.etag`: bump `last_seen_ts = int(time.time())`, do NOT re-write `snapshot_json` / `refreshed_ts`, return cached `CharacterSnapshot.model_validate_json(row.snapshot_json)`. (This is the "match" path — the cache absorbs the cost of canonicalization but skips the projection cost on hit.)
  4. Else (MISS or mismatch): project response → `CharacterSnapshot` via the allow-list projector (`_project_to_snapshot`, see below). Upsert `(character_id, snapshot_json=snapshot.model_dump_json(), etag=new_etag, last_seen_ts=now, refreshed_ts=now)`. Return the new snapshot.
  - Note: a true "no-fetch" hit (free) is impossible without a real upstream ETag tool — D-122 calls out this exact trade-off. Plan 17-02's TTL fallback gives an upper bound; without TTL the cache would still call the fetcher every time. **17-01 = baseline (every call refreshes); 17-02 = TTL short-circuits the fetcher when inside `refreshed_ts + TTL`.**
- `_project_to_snapshot(response: dict) -> CharacterSnapshot`: extract ONLY the 14 allow-list fields. Unknown fields silently dropped; missing required fields raise `ValueError` with a clear message. Combat-mutable fields in the dm20 response (e.g. `current_hp`) are **dropped** at the projector — never written, never validated against, never reach `extra="forbid"` (which would raise — we want quiet drops + fail-CLOSED on the model).
- Tests (≥6):
  - `test_get_or_fetch_miss_populates_cache` — empty repo → fetcher called → row inserted → returned model matches projection.
  - `test_get_or_fetch_etag_match_skips_write` — populate, second call with same fetcher payload → `refreshed_ts` unchanged, `last_seen_ts` advanced.
  - `test_get_or_fetch_etag_mismatch_updates` — second call with different payload → `refreshed_ts` advanced, snapshot updated.
  - `test_projector_drops_combat_state` — feed projector a dict containing `current_hp=5`, `current_conditions=["poisoned"]` alongside valid fields → output has neither key.
  - `test_projector_missing_required_raises` — drop `max_hp` from input → `ValueError`.
  - `test_projector_unknown_static_field_dropped` — feed `xp_total=12345` → silently dropped (not on allow-list).

**Done:** all `get_or_fetch` tests pass.
