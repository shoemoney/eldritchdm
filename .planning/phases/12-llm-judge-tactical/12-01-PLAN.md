# Phase 12 Plan 01 — TacticalJudge + JudgeVerdict + ScenarioEntry schema

## Objective

Ship the foundational eval primitives:

1. New package `src/eldritch_dm/eval/` (mirrors `tools/` and `observability/`).
2. `eval/judge.py` — `TacticalJudge` AsyncOpenAI wrapper + `JudgeVerdict`
   pydantic model with post-parse validator that checks
   `overall_score ≈ mean(per_dimension.values())` (D-73).
3. `eval/prompts/judge.txt` — versioned judge prompt (SemVer header on line 1)
   describing the 4 AI-SPEC §1b dimensions and anchored 0.0/0.5/1.0 rubric.
4. `eval/scenarios.py` — `ScenarioEntry` pydantic schema + `load_scenarios()`
   JSONL streaming loader (mirrors `eligibility_loader.py` fail-soft style,
   but for eval we fail LOUD on corruption — bad corpus is a developer
   bug, not an operator one).
5. `eval/__init__.py` — public exports.
6. New `traced_eval` context manager added to
   `src/eldritch_dm/observability/instrumentation.py` emitting span
   `eldritch.eval.judge` with D-81 attributes. (Scope-note: extends
   Phase 11's surface to satisfy D-81; not a deviation — D-81 explicitly
   asks for this and the existing `traced_decision` is hardwired to
   `eldritch.monster.decision`.)
7. Unit tests for: schema validation (happy + corrupt), prompt loader
   (version header parsing), judge happy path (mocked AsyncOpenAI),
   judge fallback paths (timeout, malformed JSON, dimension-mean
   validation failure).

## Critical sub-decisions

### S-12-01-A — Pydantic v2 validator shape
`JudgeVerdict` uses `@model_validator(mode="after")` (NOT a field validator),
returning `self`. Tolerance: `abs(overall_score - mean(values)) <= 0.05`.
`per_dimension` keys are the exact `Literal["tactical_intent",
"meta_knowledge", "narrative_fairness", "edge_case"]` set from D-73.
Pydantic v2 will reject extra keys naturally because `per_dimension` is
typed as a `dict[Literal[...], float]` — values outside the literal
union fail validation.

### S-12-01-B — Judge sees expected_target_pool/avoidance as author hints
CONTEXT is silent on whether the judge sees `expected_target_pool` and
`expected_avoidance` from the scenario. **Decision: yes, the judge gets
them labeled as "corpus author's expectation"**. Rationale: the judge
needs SOMETHING to anchor "would a veteran DM approve" against — otherwise
two different judges score very differently. Passing the author hint as
input (not as ground-truth-the-judge-must-match) preserves the judge's
freedom to disagree (D-73 `would_a_veteran_dm_approve: bool` is the
judge's own call, not a corpus-equality check).

### S-12-01-C — Judge fail-soft semantics
SmartMonsterDriver fails soft (D-58) because a missing target would crash
combat. The judge has no such constraint — if the judge fails, the EVAL
fails LOUD on that scenario. Specifically: `TacticalJudge.score()` returns
`JudgeVerdict | None` (None = judge failure), and the aggregator (Plan 02)
counts None results as scoring 0.0 across all dimensions with a
`judge_error` reason recorded. We do NOT silently substitute zeros — the
JSON output records `judge_error: "<reason>"` for triage.

### S-12-01-D — Versioned prompt loader
`load_judge_prompt(path)` reads the file, asserts line 1 matches
`^# judge-prompt-version: (\d+\.\d+\.\d+)$`, captures the SemVer, returns
`(prompt_text, version_str)`. Corrupt header → `JudgePromptError`. Eval
runs (Plan 02) write the version into the JSON output for reproducibility
(D-72).

### T-12-01-02 — Judge prompt + version loader

**Files:**
- `src/eldritch_dm/eval/prompts/__init__.py` (new, empty package marker)
- `src/eldritch_dm/eval/prompts/judge.txt` (new, ~50 lines)
- `src/eldritch_dm/eval/judge_prompt.py` (new — small loader module)
- `tests/eval/test_judge_prompt.py` (new)

**Behavior:**
- `judge.txt` line 1: exactly `# judge-prompt-version: 1.0.0`.
- Prompt content covers:
  - Role: "tactical combat critic for D&D 5e."
  - Input shape: `scenario` + `driver_choice` + `driver_rationale` (all explicit).
  - The 4 dimensions with 0.0/0.5/1.0 anchors each. Example:
    - `tactical_intent`: 0.0 = "ignores INT-appropriate tactics (e.g., low-INT brute picks lowest-HP foe across the map)"; 0.5 = "picks a plausible target but rationale doesn't match INT"; 1.0 = "choice and rationale align with monster INT".
    - `meta_knowledge`: 0.0 = "uses class/subclass knowledge the monster shouldn't know"; 1.0 = "relies only on visible cues".
    - `narrative_fairness`: 0.0 = "always-execute-the-downed behavior or focus-fire bloodied tank"; 1.0 = "respects anti-griefing baseline".
    - `edge_case`: 0.0 = "targets invisible/full-cover PC"; 1.0 = "respects visibility/cover rules".
  - Output schema: `{"overall_score": float, "per_dimension": {...4 keys...}, "reasoning": str ≤500 chars, "would_a_veteran_dm_approve": bool}`.
  - Self-consistency directive: `overall_score` must equal the arithmetic mean of `per_dimension` values (within ±0.05).
- `load_judge_prompt(path: Path) -> tuple[str, str]`:
  - Reads file, asserts line 1 matches `^# judge-prompt-version: (\d+\.\d+\.\d+)$`.
  - Returns `(full_text, version)`. Raises `JudgePromptError` on header mismatch or unreadable file.

**Tests:**
- Real-file load: header parses, version is `"1.0.0"`, prompt text non-empty.
- Mock file missing header → `JudgePromptError`.
- Mock file with malformed version (`# judge-prompt-version: foo`) → `JudgePromptError`.

**Verification:** `pytest tests/eval/test_judge_prompt.py -v` passes.

**Commit:** `feat(12-01): add versioned judge prompt + SemVer header loader`

### T-12-01-03 — JudgeVerdict pydantic model + dimension-mean validator

**Files:**
- `src/eldritch_dm/eval/judge.py` (new — adds JudgeVerdict + skeleton TacticalJudge class)
- `tests/eval/test_judge_verdict.py` (new)

**Behavior:**
- `JudgeVerdict` model:
  - `model_config = ConfigDict(extra="ignore", frozen=True)`.
  - `overall_score: float` (Field constrained to `[0.0, 1.0]`).
  - `per_dimension: dict[Literal["tactical_intent","meta_knowledge","narrative_fairness","edge_case"], float]` (each value `[0.0, 1.0]`).
  - `reasoning: str` (max_length=500).
  - `would_a_veteran_dm_approve: bool`.
  - `@model_validator(mode="after")` checks `len(per_dimension) == 4` AND `abs(overall_score - sum(values)/4) <= 0.05`. Raises `ValueError` with a precise message on mismatch.

**Tests:**
- Happy path: `overall = 0.75`, dims `{tactical_intent:1.0, meta_knowledge:0.5, narrative_fairness:0.75, edge_case:0.75}` (mean=0.75) → validates.
- Mean-mismatch beyond tolerance: `overall = 0.5`, dims summing to mean 0.9 → ValidationError.
- Tolerance edge: `overall = 0.75`, dims mean 0.79 → ok (diff 0.04).
- Tolerance edge: `overall = 0.75`, dims mean 0.81 → fail (diff 0.06).
- Missing dimension key → ValidationError (only 3 keys provided).
- Score out of `[0,1]` → ValidationError.
- Reasoning >500 chars → ValidationError.

**Verification:** `pytest tests/eval/test_judge_verdict.py -v` passes.

**Commit:** `feat(12-01): JudgeVerdict pydantic model with dimension-mean validator`

### T-12-01-04 — TacticalJudge AsyncOpenAI wrapper

**Files:**
- `src/eldritch_dm/eval/judge.py` (extend — add TacticalJudge class)
- `tests/eval/test_tactical_judge.py` (new)

**Behavior:**
- `TacticalJudge.__init__(*, openai_client: AsyncOpenAI, model: str = "ShoeGPT", prompt_path: Path | None = None, timeout_seconds: float = 30.0)`.
  - Loads prompt at init time via `load_judge_prompt()`. Stores `(prompt_text, prompt_version)`.
  - Default `prompt_path` is `Path(__file__).parent / "prompts" / "judge.txt"`.
- `prompt_version` exposed as a public attribute for the eval CLI to write into the JSON output.
- `async score(scenario: ScenarioEntry, driver_choice_pc_id: str, driver_rationale: str | None) -> JudgeVerdict | None`:
  - Builds a user message containing JSON: `{"scenario": {...slim view of scenario including expected_target_pool and expected_avoidance...}, "driver_choice": {"target_pc_id": ..., "rationale": ...}}`.
  - Calls AsyncOpenAI exactly like SmartMonsterDriver does (D-71): `response_format={"type":"json_object"}`, `temperature=0.0`, `max_tokens=600`, `asyncio.wait_for(..., timeout=self._timeout_seconds)`.
  - Parses content with `JudgeVerdict.model_validate_json(...)`. On `ValidationError`/`JSONDecodeError`/`TimeoutError`/any `Exception`: structured log + return `None`.
  - Token usage extraction defensive (mirror SmartMonsterDriver lines 400-406).

**Tests (`tests/eval/test_tactical_judge.py`, with `respx`-style mocked AsyncOpenAI):**
- Happy path: mock client returns valid JSON → `score()` returns a `JudgeVerdict`.
- Timeout: mock client raises `TimeoutError` → `None`.
- Malformed JSON: mock returns prose → `None` (no last-chance regex for judge — strict).
- Dimension-mean violation: mock returns JSON with `overall=0.5`, dims mean 0.9 → `None`.
- Missing dimension key: mock returns JSON with 3 keys → `None`.
- Refusal (empty content): mock returns `choices[0].message.content = ""` → `None`.
- Token usage None defensive: mock returns `usage=None` → no AttributeError; judge still returns a valid verdict.

**Verification:** `pytest tests/eval/test_tactical_judge.py -v` passes.

**Commit:** `feat(12-01): TacticalJudge AsyncOpenAI wrapper with fail-soft to None`
