# Plan 16-02 — Invalidation API + schema-version polling + OTel KPI integration

## Objective

Extend `MCPCache` (from 16-01) with: (1) public `invalidate(tool=None, args=None)` API that clears matching entries from BOTH layers, (2) an opt-in background task `start_schema_version_poller(client, interval_s=60)` that calls `dm20__schema_version` every 60s and full-wipes the cache on value change (gracefully disables itself if the tool does not exist), and (3) Phase 11/13-style OTel KPI emission for hit rate, layer sizes, and invalidation count.

## Locked design decisions

- **D-114 invalidation triggers:**
  - Explicit: `await cache.invalidate(tool_name=None, args=None)`:
    - `tool_name=None, args=None` → wipe both layers entirely.
    - `tool_name="X", args=None` → wipe all entries for tool X.
    - `tool_name="X", args={...}` → wipe single entry.
  - Auto: `start_schema_version_poller(client, interval_s=60)` calls `client.call("dm20__schema_version")` every 60s. On first call → store the version. On subsequent calls → compare; on change → call `invalidate()` (full wipe) and log a structured `mcp_cache_schema_version_changed` event.
  - **Graceful degradation:** if `dm20__schema_version` raises `MCPToolError` (4xx) the very first time, log `mcp_cache_schema_poller_disabled` and stop the poll loop. Do NOT crash startup.
- **D-115 KPIs (via Phase 11 OTel + Phase 13 span buffer):**
  - Add a new span name `eldritch.mcp.cache` (analogous to `eldritch.monster.decision`) emitted from every cacheable `call(...)`. Attributes:
    - `eldritch.mcp.tool_name` (str)
    - `eldritch.mcp.cache.layer` (`"l1" | "l2" | "miss" | "bypass"`)
    - `eldritch.mcp.cache.size_l1` (int — current L1 size)
    - `eldritch.mcp.cache.size_l2` (int — current L2 size; -1 when L2 disabled)
    - `eldritch.mcp.cache.latency_ms` (int — wall-clock inside `call()`)
  - Add a separate `traced_mcp_cache_invalidation` context manager that emits one span per invalidation with attributes:
    - `eldritch.mcp.cache.invalidation.scope` (`"all" | "tool" | "entry" | "schema_version"`)
    - `eldritch.mcp.cache.invalidation.tool_name` (str | None)
    - `eldritch.mcp.cache.invalidation.entries_removed` (int)
  - Module-level counter `_CACHE_INVALIDATIONS_TOTAL: int = 0` for tests and a public `MCPCache.metrics_snapshot()` returning a small `MCPCacheMetrics` pydantic model (`hits_l1, misses_l1, hits_l2, misses_l2, bypass_count, size_l1, size_l2, invalidations_total`).
  - All emission paths honor `OBSERVABILITY_ENABLED` — when off, the `traced_*` context manager records to the span buffer (primary sink) but skips OTel, mirroring `instrumentation.py`.

  ```python
  async def _poll():
      try:
          last = (await client.call("dm20__schema_version")).get("version")
      except MCPToolError:
          log.info("mcp_cache_schema_poller_disabled", reason="tool_not_available")
          return
      while not self._poller_stop.is_set():
          try:
              await asyncio.wait_for(self._poller_stop.wait(), timeout=interval_s)
              return  # stop event set
          except asyncio.TimeoutError:
              pass
          try:
              current = (await client.call("dm20__schema_version")).get("version")
          except Exception as exc:
              log.warning("mcp_cache_schema_poll_failed", error=str(exc)[:200])
              continue
          if current != last:
              removed = await self.invalidate()
              log.info("mcp_cache_schema_version_changed",
                       old=last, new=current, entries_removed=removed)
              last = current
  ```
  The stop signal is an `asyncio.Event` so `stop_schema_version_poller()` can wake the task immediately.</what>
