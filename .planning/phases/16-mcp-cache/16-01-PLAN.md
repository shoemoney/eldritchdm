# Plan 16-01 — MCPCache scaffolding (L1 LRU + L2 SQLite + cacheable allow-list)

## Objective

Introduce `src/eldritch_dm/mcp/cache.py` exposing an `MCPCache` class that **composes** `MCPClient` (NOT inherits). `MCPCache.call(tool_name, **arguments)` mirrors `MCPClient.call(...)` exactly so it is a drop-in. Two cache layers: in-process L1 (OrderedDict + asyncio.Lock + TTL) and an opt-in L2 (aiosqlite WAL at `~/.eldritch/mcp_cache.sqlite`). A fail-CLOSED allow-list gates which tools are cacheable.

Invalidation, schema-version polling, and OTel KPI emission are deferred to Plan 16-02.

## Locked design decisions (carry over from CONTEXT)

- **D-110** L1 = `OrderedDict` + `asyncio.Lock`. Key = `(tool_name, args_hash)`. `maxsize=512`, `ttl=300s`.
- **D-111** L2 = aiosqlite WAL at `~/.eldritch/mcp_cache.sqlite`. Schema:
  ```
  CREATE TABLE mcp_cache_entries (
    tool_name   TEXT NOT NULL,
    args_hash   TEXT NOT NULL,
    response_json TEXT NOT NULL,
    etag        TEXT,
    created_ts  INTEGER NOT NULL,
    PRIMARY KEY (tool_name, args_hash)
  );
  CREATE INDEX idx_mcp_cache_created ON mcp_cache_entries(created_ts);
  ```
- **D-112** `args_hash = sha256(json.dumps(args, sort_keys=True, separators=(',', ':'), default=str)).hexdigest()`. NEVER `hash(frozenset(...))` (PYTHONHASHSEED-random).
- **D-113** `MCPCACHE_ENABLED=true` default for L1. `MCPCACHE_L2_ENABLED=false` default (opt-in).
- **D-116** Composition, not inheritance.
- **D-117 (HARD CONSTRAINT — mechanical honesty)** Allow-list is **strictly limited to static reference data**. NO mutable-state reads in v1.5. Specifically:

  **Cacheable allow-list (final, derived from `TOOL_TO_FUNCTION`):**
  - `dm20__get_class_info` — D&D 5e class reference data
  - `dm20__get_race_info` — D&D 5e race reference data
  - `dm20__list_campaigns` — campaign roster (semi-static)
  - `dm20__get_campaign_info` — campaign metadata (semi-static)
  - `dnd__search_all_categories` — SRD rulebook search
  - `dnd__verify_with_api` — SRD verification

  **NEVER cacheable (out of scope for v1.5 — would break mechanical honesty):**
  - All `dm20__create_*`, `dm20__update_*`, `dm20__apply_*`, `dm20__set_*`, `dm20__start_*`, `dm20__end_*`, `dm20__next_*`, `dm20__remove_*`, `dm20__combat_*`, `dm20__party_*`, `dm20__player_action`, `dm20__load_*`
  - Mutable-state reads: `dm20__get_game_state`, `dm20__get_character`, `dm20__get_npc`, `dm20__get_party_status`, `dm20__list_characters`, `dm20__get_claudmaster_session_state`, `dm20__validate_character_rules`
  - Dice: `dice__dice_roll` (NEVER cache RNG)

  Any tool not on the allow-list → cache is bypassed → call goes straight to `MCPClient.call(...)`.

|-----|---------|---------|
| `MCPCACHE_ENABLED` | `true` | Master switch. When false, `MCPCache.call` always delegates straight to MCPClient. |

| `MCPCACHE_L2_PATH` | `~/.eldritch/mcp_cache.sqlite` | L2 DB file. Parent dir auto-created. |

- `aclose()` closes L2 connection if open; delegates to inner.aclose() ONLY when the caller explicitly opts in via `aclose_inner=True` (default False — composition means the inner is owned by the caller).

Lookup order on `call(tool_name, **arguments)`:
1. If `tool_name not in CACHEABLE_TOOLS` OR `settings.mcpcache_enabled is False` → straight delegate to `inner.call(...)`. Return.
2. Compute `args_hash`. Look up `(tool_name, args_hash)` in L1.
3. L1 HIT (within TTL) → move to MRU end, return cached value.
4. L1 MISS → if L2 enabled, look up in L2 (within `MCPCACHE_L2_TTL_S`).
5. L2 HIT → promote to L1 MRU, return.
6. Total MISS → `await inner.call(...)`, store in L1 (+ L2 if enabled), return.

Public properties for tests: `l1_size`, `l1_hits`, `l1_misses`, `l2_hits`, `l2_misses`, `bypass_count` (integer counters reset on `reset_counters()`). No KPI emission yet — Plan 16-02 wires those.</what>
