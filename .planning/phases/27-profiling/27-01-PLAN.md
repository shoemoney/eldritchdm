## Locked design (mid-plan choices the executor must honor)

1. **Two-runs-per-path:** cProfile adds 2-10x overhead so percentiles from a
   cProfile run are noise. The profiler does:
   - **Wall-clock run:** 100 iterations with `time.perf_counter_ns()`,
     populates `p50_ms`/`p95_ms`/`p99_ms`.
   - **cProfile run:** 20 iterations under `cProfile.Profile()`, populates
     `cprofile_top_10` (formatted `"module.func:lineno (cumtime_pct)"`).

2. **Combat-turn-resolution = inner sequence**, not full orchestrator loop.
   We profile the chain `mcp_tools.party_pop_action → party_thinking →
   party_resolve_action` against respx-mocked dm20 + AsyncMock rate_limiter.
   The orchestrator's outer `_loop` adds asyncio scheduling overhead that is
   not what players wait on; the dm20-call chain is. Documented in
   `docs/PERFORMANCE.md` measurement-methodology column.

3. **Riposte-click-handler = `handle_riposte_click`** with AsyncMock
   `interaction`, AsyncMock `warning_sender`, AsyncMock `mcp` (respx not
   needed — we call `mcp_tools` underneath through `_mcp` mock), real
   `SessionLocks`, real `RiposteTimerRepo` against a fresh tmp SQLite. Test
   data: ONE pending timer row inserted per iteration, then handler called
   for the happy-path branch (mark_consumed). Variant runs not needed —
   we profile the success path only.

4. **Character-ingest-fast-path = `translate_to_character_sheet`**, OpenAI
   client mocked with `respx` against
   `http://localhost:8080/v1/chat/completions` returning a fixed valid JSON
   body (no real LLM). Profiles sanitize + JSON parse + pydantic validate.

5. **Ingest-pipeline-ocr = `ingest()` over png_bytes** with
   `ocrmac`/`easyocr` monkeypatched to return canned regions (mirrors
   `tests/ingest/conftest.py:mock_ocrmac_regions`), and OpenAI client mocked
   via respx. PDF path is NOT separately profiled (same downstream pipeline;
   adds reportlab work that's out-of-scope for OUR-CODE measurement).

6. **Smart-driver-oracle sub-paths (3):**
   - `smart-success` — LLM mock returns valid `MonsterTacticChoice` JSON;
     cache cold (clear between iterations).
   - `smart-fallback-to-random` — LLM mock returns invalid JSON; profiler
     measures the parse-failure → random-fallback path.
   - `cache-hit` — first call populates cache, then 99 iterations hit cache.

7. **MCP-cache-roundtrip sub-paths (3):**
   - `l1-hit` — call once to populate L1, then 99 iterations hit L1.
   - `l1-miss-l2-hit` — pre-populate L2 only, evict from L1 between iter.
   - `l1-l2-miss` — clear caches between iterations; respx dm20 mock hit.

8. **JSON output validation:** A `BaselineSchema` pydantic v2 model lives in
   the script and validates the assembled output before write. The
   self-check test imports `BaselineSchema` to re-validate the committed
   `.planning/perf-baseline-v1.9.0.json`.

9. **Wallclock budget:** Profiler MUST finish in ≤120 s wallclock on M3
   Ultra. Self-check test asserts the limit by running the profiler with
   `--iterations 5` against a temp output path and checking it returns in
   <30 s — keeps regular `pytest` runs fast while still smoke-covering all
   code branches in the profiler.

### Task 2 (auto) — Measurement primitives: `measure_walltime`, `measure_cprofile`, `format_top_10`

**Files:**
- `scripts/perf/profile_hot_paths.py` (modify)

**Behavior:**
- `async def measure_walltime(label, async_fn, *, iterations) -> tuple[p50, p95, p99]`
  using `time.perf_counter_ns()` and `statistics.quantiles(method='exclusive')`
  or manual sort+index (whatever yields exact percentiles on 100 samples).
- `async def measure_cprofile(label, async_fn, *, iterations) -> list[str]`
  wraps `cProfile.Profile()`, accumulates over iterations, sorts by
  cumulative, formats top 10 as `f"{module}.{func}:{lineno} ({pct:.1f}%)"`.
- `format_top_10` uses `pstats.Stats(prof).sort_stats('cumulative')` then
  reads `prof.stats` dict directly (key = `(file, lineno, func)`,
  value tuple has `(cc, nc, tt, ct, callers)`); compute cumtime% = ct /
  total_cumtime * 100; module name = `Path(file).stem`.

**Verification:**
- Run against `asyncio.sleep(0.001)`; p50 ≈ 1ms, top_10 includes
  `asyncio.sleep`.

**Done when:** Both primitives work standalone, ruff clean.
