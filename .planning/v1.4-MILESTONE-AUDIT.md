## Executive Summary

v1.4 had a remarkable lifecycle that demonstrated the GSD honest-report
contract working under stress:

1. **First Phase 15 agent halted on premise check.** v1.3 had surfaced
   "WRITER-QUEUE-HANG-01" as the v1.4 top priority. The first Phase 15
   agent ran empirical reproduction and found the hangs DON'T reproduce
   at HEAD — and that the `WriterQueue` already implements the exact
   asyncio.Event + sentinel pattern the v1.4 CONTEXT.md proposed
   adding. Halted with a 169-line evidence-rich halt-report identifying
   the REAL bug (FLAKE-02 test-isolation in `bot_factory`).

2. **Orchestrator attempted inline fix.** Tried halt-report option (c)
   (sys.modules autouse-clear). Pytest hung in 3 separate verification
   runs of this session. Reverted untested. Marked v1.4 halt-deferred.

3. **User overrode the halt** and spawned a fresh agent with the
   rescoped FLAKE-02 scope (halt-report option a). Agent worked in
   worktree isolation (immune to the orchestrator-session hang quirk).

4. **Agent shipped the actual fix** — but discovered the cleanest
   approach isn't pure option (a). `unload_extension` in teardown alone
   is insufficient because it deletes `sys.modules[cog]` without
   restoring the orphaned pre-`load_extension` module that test files
   captured at collection time. Final fix is **snapshot+restore** in a
   suite-wide autouse fixture (`tests/conftest.py`) that captures
   `sys.modules[cog]` before each test and restores it afterward,
   combined with the unload-on-teardown in `tests/bot/conftest.py`.

The fix landed in 2 commits. Full suite ran GREEN in 2 consecutive runs
(1244 passed, 17 skipped, 0 failed). FLAKE-02 (carried from v1.3) is
closed.
